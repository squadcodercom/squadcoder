# מנהל משלוחים ישראלי

## הוראות

**קריטי: סקיל זה הוא מדריך למפתחים לבניית מערכות שילוח. אין לך גישה למערכות מעקב חבילות. אם משתמש שואל "איפה החבילה שלי" או נותן מספר מעקב לבדיקה, חובה:**
1. **להפנות אותו לאתר חברת השילוח** (דואר ישראל: mypost.israelpost.co.il, HFD: hfd.co.il, צ'יטה: chita-il.com)
2. **לעולם לא להמציא סטטוס חבילה, מיקום איסוף, שם סניף, כתובת, שעות פעילות, או כל פרט מעקב אחר.** אין לך גישה למערכת מעקב של אף חברת שילוח.
3. **אם אתם צריכים מעקב אוטומטי**, הדריכו אותו לבנות אינטגרציית מעקב (שלב 4 למטה) או המליצו על סקיל הקהילה `israel-post-tracking` לחבילות דואר ישראל.

### שלב 1: בחירת חברת שילוח בהתאם למשלוח
עזרו למשתמש לבחור את חברת השילוח המתאימה למשלוח. שאלו על גודל החבילה, משקל, דחיפות, יעד ותקציב. השתמשו בטבלת ההשוואה:

| חברת שילוח | מתאים ל... | מהירות | טווח מחירים | נקודות איסוף | אינטגרציה |
|------------|-----------|--------|-------------|--------------|-----------|
| דואר ישראל | חבילות רגילות, כיסוי ארצי | 3-5 ימי עסקים | נמוך-בינוני | סניפי דואר + לוקרי BOX2GO | Datalogics API, ספריות קוד פתוח |
| צ'יטה | משלוח מהיר/באותו יום | באותו יום (לעיתים תוך 4 שעות) | בינוני-גבוה | מוגבל | אפליקציית Shopify, API עסקי פרטי |
| HFD (המריץ ופלאש) | הפצה למסחר אלקטרוני | 2-4 ימי עסקים | בינוני | כ-1,000 נקודות איסוף + לוקרים | תוספי Shopify/WooCommerce, API פרטי |
| מהיר לי | שליחויות B2B באותו יום | באותו יום (תוך 9 שעות) | בינוני-גבוה | דלת לדלת בלבד | דרך פלטפורמת LionWheel |
| UPS ישראל (לוקרים) | חבילות קטנות, תעריף אחיד | 1-2 ימי עסקים | כ-27 ש"ח כולל מע"מ | 100 לוקרים + 150 חנויות שירות בפריסה ארצית (השקה מרץ 2025) | UPS Developer Kit + הפקדה בלוקר עם קוד QR |
| GetPackage (getpackage.com) | שליחות באותו יום מבוססת כלכלת שיתוף | באותו יום (לרוב 1-3 שעות) | משתנה (לפי הצעת מחיר) | דלת לדלת בלבד | פלטפורמת web + REST API לחשבונות עסקיים |

חברות שליחויות נוספות משתמשות בתוכנת ניהול המשלוחים בלדר (תפוז, חפוז, יסגב, מאך1). אם המשתמש מזכיר "בלדר", בררו איזו חברת שליחויות הוא מתכוון, כי בלדר היא תוכנה ולא חברת שילוח.

שירותי לוקרים ונקודות איסוף (לא חברות שילוח עצמאיות):

| שירות | מפעיל | מיקומים | הערות |
|-------|-------|---------|-------|
| BOX2GO | דואר ישראל + פז | כ-120 לוקרי Yellow Box בתחנות פז | משולב בשילוח דואר ישראל |
| שלאגר | אוריין | לוקרים חכמים ניידים באזורי מגורים | רשת לוקרים אוטומטית |
| Done | Done (done.co.il) | לוקרים במיקומים שונים | כ-30 ש"ח לחבילה, עד 5 ק"ג, 3 ימי עסקים |
| SafeLocker | SafeLocker | קניונים ומרכזי קניות | איסוף מלוקר |
| נקודות HFD | HFD | כ-1,000 ברחבי הארץ | חלק מרשת ההפצה של HFD |
| לוקרים של UPS ישראל | זכיין UPS ישראל | 100 לוקרים + 150 חנויות שירות | הפקדה ואיסוף 24/7, כ-27 ש"ח לחבילה (השקה מרץ 2025) |

קריטריונים לבחירה:
- **מהירות משלוח:** משלוח באותו יום דורש צ'יטה, מהיר לי או GetPackage; UPS ישראל ב-1-2 ימי עסקים; משלוח רגיל מתאים לכל חברות השילוח
- **גודל ומשקל:** חבילות כבדות או חריגות בגודל מגבילות את האפשרויות. לשירותי לוקרים יש מגבלות גודל (Done עד 5 ק"ג, לוקרי UPS לחבילות קטנות בלבד).
- **אזור יעד:** המרכז (גוש דן) בכיסוי מלא; פריפריה (אילת, גליל) עם כיסוי מוגבל בחלק מחברות השילוח
- **תקציב:** דואר ישראל הכי זול למשלוח רגיל; UPS ישראל מציע תעריף אחיד של כ-27 ש"ח מלוקר ללוקר; צ'יטה ומהיר לי פרימיום למהירות
- **איסוף מול דלת לדלת:** BOX2GO/HFD/Done/לוקרי UPS להפקדה ואיסוף עצמי; צ'יטה/מהיר לי/GetPackage למשלוח עד הדלת
- **נפח:** מהיר לי דורש מינימום 10 משלוחים ביום; דואר ישראל, HFD, UPS ישראל ו-GetPackage ללא מינימום

### שלב 2: עיצוב כתובת ישראלית
עצבו כתובות בהתאם לדרישות חברת השילוח. הריצו `python scripts/format_address.py --validate` לבדיקת פורמט לפני שליחה. עיינו ב-references/address-format.md למפרט המלא.

שדות נדרשים:
- **שם רחוב** -- בעברית
- **מספר בית**
- **דירה** -- אם רלוונטי
- **כניסה** -- אם רלוונטי (נפוץ בבניינים ישנים)
- **קומה** -- אם רלוונטי
- **עיר** -- בעברית
- **מיקוד** -- 7 ספרות

טיפול מיוחד:
- **כתובות צבאיות:** השתמשו בפורמט כתובת צה"ל (מספר צבאי). רק דואר ישראל מטפל בדואר צבאי.
- **קיבוצים/מושבים:** שם הישוב + מספר בית (ללא רחוב)
- **אזורי תעשייה:** שם האזור + שם בניין/חברה
- **יישובים ערביים:** ודאו שהתעתיק תואם למאגר של חברת השילוח

### שלב 3: הפקת תווית משלוח
הפקת תוויות תלויה בשיטת האינטגרציה של חברת השילוח:

- **דואר ישראל:** השתמשו ב-Datalogics API (`connect.datalogics.co.il/rest/w_create_shipping`) או בפורטל העסקי של דואר ישראל (mybusiness.israelpost.co.il)
- **צ'יטה:** השתמשו באפליקציית "Cheetah DeliverIt" ב-Shopify, או צרו קשר עם מכירות צ'יטה לאינטגרציה ישירה
- **HFD:** השתמשו באפליקציית "HFD DeliverIt" ב-Shopify או בתוסף "HFD ePost Integration" ל-WooCommerce. ל-HFD גם API פרטי לאינטגרציה ישירה (צרו קשר עם HFD).
- **מהיר לי:** אינטגרציה דרך פלטפורמת LionWheel (lionwheel.com). צרו קשר עם מהיר לי לפתיחת חשבון עסקי.
- **UPS ישראל:** השתמשו ב-UPS Developer Kit (developer.ups.com) להפקת תוויות, חישוב תעריפים ומעקב. לתהליך הפקדה בלוקר, הפיקו תווית והציגו את קוד ה-QR בכל לוקר UPS (24/7). הרשת כוללת 100 לוקרים ו-150 חנויות שירות בפריסה ארצית.
- **GetPackage:** השתמשו בפלטפורמה העסקית של GetPackage (getpackage.com). לאינטגרציות בנפח גבוה, בקשו גישת REST API מצוות העסקים שלהם.

כל התוויות דורשות:
- פרטי שולח (שם, כתובת, טלפון)
- פרטי נמען (שם, כתובת, טלפון)
- מידות ומשקל החבילה
- סוג שירות (רגיל, מהיר, רשום)
- ברקוד מעקב

ראו references/carrier-apis.md לפרטי אינטגרציה לכל חברת שילוח.

### שלב 4: הגדרת מעקב חוצה-חברות

**חשוב:** שלב זה עוסק בבניית מערכת מעקב לאפליקציה שלכם, לא בבדיקת סטטוס של חבילה ספציפית. אם המשתמש רוצה לבדוק חבילה עכשיו, הפנו אותו לאתר חברת השילוח (ראו ההערה הקריטית בתחילת ההוראות).

אף חברת שילוח ישראלית לא מציעה REST API ציבורי מתועד למעקב. הטמיעו מעקב אחיד באחת מהגישות הבאות:

**אפשרות א: צובר צד שלישי (מומלץ)**
השתמשו ב-AfterShip, TrackingMore או WeShip למעקב אחיד מול כל חברות השילוח. אלה מספקים REST API מתועד עם webhooks.

**אפשרות ב: אינטגרציה ישירה לכל חברה**
- דואר ישראל: גרידת מעקב מ-`mypost.israelpost.co.il` (דורש טיפול בטוקן CSRF) או שימוש בספריות קוד פתוח (`bennymeg/IsraelPostalServiceAPI`, `LandRover/postil-status`)
- HFD: מעקב דרך hfd.co.il או AfterShip
- צ'יטה: מעקב דרך מערכת RUN (chita-il.com) או אפליקציית Shopify
- מהיר לי: מעקב דרך LionWheel

נרמלו קודי סטטוס לסט אחיד: `pending`, `picked_up`, `in_transit`, `out_for_delivery`, `delivered`, `failed_delivery`, `returned`

בצעו פולינג במרווחים מוגדרים (ברירת מחדל: כל שעתיים). זהו חריגות: חבילה תקועה באותו סטטוס מעל 48 שעות, כשלי מסירה, תיקוני כתובת.

### שלב 5: הגדרת התראות ללקוח
הגדירו התראות אוטומטיות ללקוחות בעת שינוי סטטוס. לשליחת SMS בעברית למספרים ישראליים, השתמשו בשערי SMS מקומיים (019/טלזר, Cellact, InforuMobile, SMS4Free) ולא ב-Twilio. SMS בעברית מוגבל ל-70 תווים לסגמנט (לעומת 160 בלטיני), ושערים מקומיים מטפלים בניתוב לרשתות הישראליות באמינות גבוהה יותר. ראו את הסקיל `israeli-sms-gateway` לפרטי אינטגרציה ספציפיים ל-SMS.

התראות לפי סטטוס:
- **נשלח:** WhatsApp/SMS עם מספר מעקב וצפי הגעה -- "החבילה שלך נשלחה! מספר מעקב: [X]. צפי הגעה: [DATE]."
- **בדרך למסירה:** "החבילה שלך בדרך אליך! צפי הגעה היום עד [TIME]."
- **נמסר:** "החבילה נמסרה בהצלחה! תודה על הקנייה."
- **כשל מסירה:** "לא הצלחנו למסור את החבילה. נסיון נוסף מתוכנן ל-[DATE]. לתיאום: [PHONE]."
- **מוכן לאיסוף:** "החבילה שלך מחכה לך בנקודת [BOX2GO/HFD/Done] ב-[LOCATION]. קוד איסוף: [CODE]."

כבדו שעות שקט: ללא התראות בין 22:00-08:00 שעון ישראל.

### שלב 6: טיפול בהחזרות ו-RMA
נהלו משלוחי החזרה בהתאם לתקנות הגנת הצרכן (עסקת מכר מרחוק), חוק הגנת הצרכן:
- הפיקו תווית החזרה עם הפניה למספר מעקב מקורי
- עקבו אחר משלוח ההחזרה חזרה למוכר
- תמכו בסיבות החזרה שונות: פגום, פריט שגוי, שינוי דעת. ללקוח 14 יום מקבלת המוצר לבטל עסקת מכר מרחוק (רכישה אונליין/בטלפון) ללא צורך בנימוק.
- **מועד החזר הכספי:** המוכר חייב להחזיר את הכסף באותו אמצעי תשלום ששימש לרכישה, תוך 7 ימי עסקים מהיום שבו ההחזרה התקבלה.
- **דמי ביטול:** בהחזרה מסיבה של "שינוי דעת" המוכר רשאי לגבות דמי ביטול בגובה 5% ממחיר המוצר או 100 ש"ח, הנמוך מביניהם. אסור לגבות דמי ביטול כשהמוצר פגום, לא תואם לתיאור, סופק באיחור, או בכל הפרה אחרת מצד המוכר.
- **חריגים:** זכות הביטול תוך 14 יום לא חלה על מוצרים מתכלים, פריטים מותאמים אישית, הלבשה תחתונה, ורהיטים שהורכבו כבר אצל הלקוח.
- חשבו עלות משלוח החזרה לפי חברת שילוח (הפקדה בלוקר UPS לרוב הזולה ביותר לחבילות קטנות בכ-27 ש"ח אחיד)
- עדכנו סטטוס הזמנה כשההחזרה מתקבלת

## דוגמאות

### דוגמה 1: מוכר מסחר אלקטרוני שולח 50 חבילות במשלוח מעורב
המשתמש אומר: "אני צריך לשלוח 50 הזמנות היום, בגדלים שונים. חלק צריכים להגיע מחר לתל אביב, השאר משלוח רגיל לכל הארץ."
פעולות:
1. ניתוח הזמנות לפי גודל, יעד ודחיפות
2. ניתוב הזמנות מהירות לתל אביב לצ'יטה דרך אפליקציית Shopify שלהם (12 חבילות)
3. ניתוב חבילות רגילות לדואר ישראל או HFD (38 חבילות) לפי עלות וזמינות תוסף
4. עיצוב כל הכתובות ואימות מיקודים דרך `scripts/format_address.py`
5. הפקת תוויות משלוח לפי שיטת האינטגרציה של כל חברת שילוח
6. הגדרת מעקב אחיד דרך AfterShip והגדרת התראות WhatsApp
תוצאה: 50 תוויות משלוח הופקו מ-3 חברות שילוח. לוח מעקב הוגדר עם התראות WhatsApp לכל לקוח.

### דוגמה 2: הגדרת אינטגרציה עם נקודות איסוף HFD לחנות Shopify
המשתמש אומר: "אני רוצה להציע איסוף מנקודות HFD כאופציה בחנות המקוונת שלי ב-Shopify."
פעולות:
1. התקנת אפליקציית "HFD DeliverIt" מחנות האפליקציות של Shopify
2. הגדרת פרטי חשבון עסקי HFD באפליקציה
3. הפעלת נקודות איסוף HFD כאפשרות משלוח בקופה
4. הגדרת התראות מעקב: לקוחות מקבלים SMS כשהחבילה מגיעה לנקודת איסוף
5. הגדרת מעקב אוטומטי מהשילוח ועד איסוף הלקוח
תוצאה: אינטגרציית HFD פעילה. לקוחות יכולים לבחור מבין כ-1,000 נקודות איסוף של HFD בקופה ולקבל התראות איסוף.

### דוגמה 3: השוואת מחירים לחבילה כבדה לאילת
המשתמש אומר: "אני צריך לשלוח חבילה של 15 ק"ג לאילת, מה האפשרויות שלי?"
פעולות:
1. בדיקת זמינות חברות שילוח לאילת (אזור פריפריה)
2. שימוש במחשבון תעריפי דואר ישראל (ציבורי, ללא הרשמה) דרך ספריית `bennymeg/IsraelPostalServiceAPI`
3. פנייה ל-HFD לקבלת הצעת מחיר (אין API ציבורי לתעריפים)
4. הערה: מהיר לי דורשים מינימום 10 משלוחים ביום, כנראה לא מתאים לחבילה בודדת
5. לצ'יטה יש סניף באילת אך במחיר פרימיום
6. השוואת זמני אספקה ועלויות
תוצאה: דואר ישראל הוא האפשרות הנגישה ביותר עם חישוב תעריפים ציבורי. לתעריפים טובים יותר בנפחים, פנו ישירות ל-HFD.

## משאבים מצורפים

### חומרי עזר
- `references/carrier-apis.md` -- שיטות אינטגרציה מאומתות לכל חברת שילוח ישראלית: דואר ישראל (Datalogics API, ספריות קוד פתוח), צ'יטה (אפליקציית Shopify, מערכת RUN), HFD (תוספי Shopify/WooCommerce), מהיר לי (LionWheel), שירותי לוקרים, וצוברי צד שלישי. היעזרו בקובץ זה בעת אינטגרציה עם חברת שילוח ספציפית בשלבים 3-4.
- `references/address-format.md` -- מפרט עיצוב כתובות ישראליות: רחוב, בית, דירה, כניסה, קומה, עיר, מיקוד. כולל פורמטים מיוחדים לקיבוצים, כתובות צבאיות ואזורי תעשייה. היעזרו בקובץ זה בעת עיצוב כתובות בשלב 2.

### סקריפטים
- `scripts/format_address.py` -- מאמת ומעצב כתובות משלוח ישראליות בהתאם לדרישות חברות השילוח. בודק תקינות מיקוד, מנרמל טקסט עברי, ומטפל בסוגי כתובות מיוחדים (קיבוץ, צבאי, אזור תעשייה). הרצה: `python scripts/format_address.py --help`

## מלכודות נפוצות

- **לעולם לא להמציא תוצאות מעקב חבילות.** אין לך גישה למערכת מעקב של אף חברת שילוח. אם משתמש נותן מספר מעקב ושואל "איפה החבילה שלי", אסור להמציא סטטוס, מיקום איסוף, שם סניף, כתובת, או שעות פעילות. סוכנים שממציאים נתוני מעקב מספקים מידע שגוי ומסוכן (למשל, לומר למשתמש שהחבילה ממתינה בסניף מסוים כשבפועל היא כבר נמסרה לעיר אחרת). תמיד הפנו את המשתמש לדף המעקב הרשמי של חברת השילוח.
- חברות שילוח ישראליות לא מציעות REST API ציבורי מתועד. אינטגרציה מתבצעת דרך תוספי פלטפורמה (Shopify, WooCommerce), הסכמים עסקיים פרטיים, או צוברי צד שלישי. סוכנים עלולים לנסות לקרוא לנקודות קצה API בדויות שלא קיימות.
- אזורי חלוקה של דואר ישראל שונים מאזורים גיאוגרפיים. הערכות זמני משלוח צריכות להשתמש במיפוי אזורי דואר ישראל, לא חישובי מרחק קו ישר.
- כתובות ישראליות לא משתמשות במיקוד בפורמט אמריקאי. מיקוד ישראלי הוא 7 ספרות. סוכנים עלולים לבדוק מול דפוסי ZIP בני 5 ספרות.
- משלוחים ביום שישי בישראל מסתיימים עד שעות הצהריים המוקדמות (לפני שבת). שירותי משלוח באותו יום לא פועלים משישי אחר הצהריים עד מוצאי שבת.
- כתובות ביישובים ביהודה ושומרון דורשות טיפול מיוחד במשלוחים ולא כל חברות השילוח מכסות אותן. ודאו כיסוי חברת שילוח לאזורים אלה.
- תשלום במסירה (COD) עדיין נפוץ במסחר אלקטרוני ישראלי. סוכנים עלולים לא לכלול אפשרות תשלום זו בהגדרת תהליכי משלוח.
- בלדר היא תוכנת ניהול משלוחים, לא חברת שילוח. אם משתמש אומר "אני שולח עם בלדר", הוא מתכוון לחברת שליחויות שמשתמשת בבלדר (תפוז, חפוז, יסגב, מאך1). בררו איזו חברה ספציפית.
- אתר דואר ישראל משתמש בהגנת בוטים (ShieldSquare/PerimeterX). גרידה אוטומטית של מעקב או תעריפים עלולה להיחסם. השתמשו בספריות קוד פתוח או ב-Datalogics API במקום.
- **רף הפטור ממס לייבוא אישי הוא 130 דולר, לא 150 דולר.** הרף הועלה ל-150 דולר ב-2025 ואז הוחזר ל-130 דולר אחרי לחץ של הקמעונאים הישראלים. הזמנות בשווי 130 דולר ומעלה (שווי הסחורה ללא משלוח/ביטוח) חייבות במע"מ 18% ועלולות לחייב גם מכס. אם המשתמש שואל על עלות שילוח חוצה גבולות או מכס, הפנו אותו לסקיל `israeli-customs-duty-calculator` -- הסקיל הזה מיועד למשלוחים מקומיים בלבד.
- **רשת הלוקרים של UPS ישראל חדשה (השקה מרץ 2025).** מסמכי שילוח ישנים יותר לא יזכירו אותה. הרשת כוללת 100 לוקרים ו-150 חנויות שירות בפריסה ארצית, תעריף אחיד של כ-27 ש"ח לחבילה כולל מע"מ, אספקה ב-1-2 ימי עסקים. זה לא אותו שירות כמו UPS Express הבינלאומי.

## פתרון בעיות

### שגיאה: "Invalid mikud (ZIP code)"
סיבה: מיקוד ישראלי חייב להיות בדיוק 7 ספרות ולתאום לשילוב עיר/רחוב.
פתרון: ודאו את המיקוד בחיפוש מיקוד של דואר ישראל (mypost.israelpost.co.il/zipcodesearch). בעיה נפוצה: מיקודים ישנים בני 5 ספרות -- כל המיקודים בישראל הם כיום בני 7 ספרות.

### שגיאה: "Carrier API endpoint not found" או "404 on API call"
סיבה: לחברות שילוח ישראליות אין REST API ציבורי מתועד. נקודת הקצה שאתם קוראים לה כנראה לא קיימת.
פתרון: בדקו ב-references/carrier-apis.md את שיטת האינטגרציה הנכונה. השתמשו בתוספי פלטפורמה (Shopify, WooCommerce), ב-Datalogics API לדואר ישראל, או בצובר צד שלישי (AfterShip, WeShip).

### שגיאה: "Address not recognized by carrier"
סיבה: פורמט הכתובת לא תואם למאגר של חברת השילוח, או בעיית קידוד טקסט עברי.
פתרון: ודאו שהכתובת בקידוד UTF-8 עברית. הריצו `scripts/format_address.py --validate` לבדיקת פורמט. ליישובים ערביים, ודאו שחברת השילוח מקבלת את האיות הספציפי. נסו תעתיקים חלופיים.

### שגיאה: "Delivery failed -- recipient not found"
סיבה: נפוץ בבנייני מגורים ללא אינטרקום או עם פרטי כניסה/קומה חסרים.
פתרון: הוסיפו כניסה וקומה לכתובת. הגדירו התראת מסירה שכוללת טלפון נמען ליצירת קשר עם השליח. שקלו מעבר לנקודת איסוף (HFD/BOX2GO) לכתובות עם כשלי מסירה חוזרים.

### שגיאה: "Tracking data not updating"
סיבה: נקודת קצה המעקב של דואר ישראל דורשת טוקן CSRF שפג תוקפו. הגנת בוטים עלולה לחסום בקשות אוטומטיות.
פתרון: השתמשו בספריות קוד פתוח `LandRover/postil-status` או `bennymeg/IsraelPostalServiceAPI` שמטפלות ברענון הטוקן. לחלופין, השתמשו בצובר צד שלישי כמו AfterShip למעקב אמין.

### משתמש שואל: "איפה החבילה שלי?" או מספק מספר מעקב
סיבה: המשתמש רוצה בדיקת סטטוס בזמן אמת, לא מדריך אינטגרציית שילוח.
פתרון: סקיל זה לא יכול לבדוק מעקב חבילות בזמן אמת. הפנו את המשתמש לדף המעקב הרשמי של חברת השילוח: דואר ישראל (mypost.israelpost.co.il), HFD (hfd.co.il), או צ'יטה (chita-il.com). למעקב אוטומטי אחרי חבילות דואר ישראל, המליצו על סקיל הקהילה `israel-post-tracking`. לעולם לא לנחש או להמציא סטטוס חבילה.
