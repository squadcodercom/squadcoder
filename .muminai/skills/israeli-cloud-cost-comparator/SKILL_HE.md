# השוואת עלויות ענן בישראל

## הוראות

### שלב 1: הבנת דרישות הענן של המשתמש

אסוף את המידע הבא לפני השוואת עלויות:

1. **סוג העומס**: אפליקציית ווב, backend של API, צינור נתונים, אימון ML, אתר סטטי, מסד נתונים
2. **היקף**: תעבורה צפויה (בקשות/חודש), אחסון נתונים (GB/TB), צרכי מחשוב (vCPU/RAM)
3. **דרישות ציות**: האם הנתונים חייבים להישאר בישראל? האם יש דרישות רגולטוריות (רשות הגנת הפרטיות, GDPR למשתמשים אירופיים)?
4. **תקציב**: תקציב חודשי בשקלים או דולרים, העדפה לתשלום לפי שימוש מול מחויבות
5. **מחסנית טכנולוגית**: שפה/פריימוורק, סוג מסד נתונים, העדפה לקונטיינרים או serverless
6. **מסלול צמיחה**: סטארטאפ (צמיחה מהירה), עסק קטן-בינוני (יציב), או ארגון (צפוי)
7. **קרדיטים ומבצעים קיימים**: שאל במפורש האם למשתמש יש קרדיטים פעילים, שכבה חינמית, מבצעים או תוכניות הנחה אצל ספק ענן כלשהו. זה כולל: הטבות שכבה חינמית (למשל AWS Free Tier עם מגבלות לפי שירות, GCP $300 ניסיון, Azure $200 ניסיון), קרדיטים מתוכניות סטארטאפ (AWS Activate, GCP for Startups, Microsoft Founders Hub), קרדיטים לסטודנטים (GitHub Student Pack), קרדיטים מהאקתונים או אירועים, קרדיטים מקרנות הון סיכון או מאיצים, או כל יתרה פרומוציונאלית אחרת. קרדיטים קיימים יכולים לשנות את ההשוואה לחלוטין וחובה לשקלל אותם בהמלצה.

### שלב 2: השוואת אזור AWS ישראל (il-central-1)

AWS השיקה את אזור ישראל (תל אביב) `il-central-1` באוגוסט 2023. פרטים עיקריים:

**שירותים זמינים ב-il-central-1:**
- EC2 (מחשוב), EBS (אחסון בלוקים), S3 (אחסון אובייקטים)
- RDS (מסדי נתונים מנוהלים: PostgreSQL, MySQL, Aurora)
- Lambda (serverless), ECS/EKS (קונטיינרים)
- ElastiCache (Redis/Memcached), DynamoDB
- CloudFront (CDN עם נקודת קצה בתל אביב), Route 53

**מדדי תמחור (il-central-1 מול eu-west-1 אירלנד):**
- מכונות EC2 יקרות יותר ב-5%-15% בדרך כלל מ-eu-west-1
- אחסון S3 במחיר דומה
- העברת נתונים החוצה באותו תמחור גלובלי
- מכונות RDS נושאות פרמיה דומה של 5%-15%

**מתי להשתמש ב-il-central-1:**
- דרישות תושבות נתונים מחייבות אחסון ישראלי
- אפליקציות רגישות להשהיה המשרתות משתמשים ישראליים (1-3 מ"ש מקומי מול 40-60 מ"ש ל-eu-west-1)
- שירותים פיננסיים, בריאות או אפליקציות ממשלתיות
- ציות לתקנות רשות הגנת הפרטיות הישראלית

**מתי eu-west-1 עדיף:**
- אין דרישות תושבות נתונים ועלות היא השיקול העיקרי
- אפליקציות המשרתות משתמשים ישראליים ואירופיים כאחד
- זמינות שירותים רחבה יותר (שירותי AWS חדשים מושקים ב-eu-west-1 לפני il-central-1)

**כתובת תמחור:** `https://aws.amazon.com/ec2/pricing/on-demand/` (סנן לפי אזור: Israel)

### שלב 3: השוואת Google Cloud Platform (me-west1)

אזור `me-west1` של GCP ממוקם בתל אביב, נפתח ב-2022 והגיע לזמינות כללית בנובמבר 2022. זהו האזור הראשון של GCP במזרח התיכון.

**שירותים זמינים ב-me-west1:**
- Compute Engine, Cloud Storage, Cloud SQL
- GKE (Kubernetes), Cloud Run (קונטיינרים serverless)
- Cloud Functions, Pub/Sub, BigQuery
- Memorystore (Redis), Cloud Spanner

**מדדי תמחור:**
- Compute Engine בדרך כלל 5%-10% זול יותר ממכונת EC2 מקבילה ב-il-central-1 של AWS
- תמחור Cloud Storage תחרותי עם S3
- הנחות שימוש מתמשך (sustained use) חלות אוטומטית (עד 30% עבור מכונות שרצות 100% מהחודש)
- הנחות שימוש מחויב: שנה (37% הנחה) או 3 שנים (55% הנחה) לעומסים צפויים

**יתרונות GCP למפתחים ישראליים:**
- BigQuery זמין ב-me-west1 (חשוב לאנליטיקת נתונים עם מידע ישראלי)
- Cloud Run מציע שכבה חינמית נדיבה (2 מיליון בקשות/חודש)
- אחסון Firebase עם backend ב-me-west1 מספק אחסון full-stack בהשהיה נמוכה
- תוכנית GCP for Startups פעילה בישראל (ראה שלב 7)

**כתובת תמחור:** `https://cloud.google.com/compute/vm-instance-pricing` (סנן לפי אזור: me-west1)

### שלב 4: השוואת Microsoft Azure

Azure משרתת את ישראל בעיקר דרך האזורים הבאים:

**אזורים:**
- **Israel Central** (הושק ב-12 בנובמבר 2023): אזור Azure מלא בישראל
- **West Europe** (הולנד): חלופה עם קטלוג שירותים רחב יותר

**שירותים זמינים ב-Israel Central:**
- Virtual Machines, Azure Blob Storage, Azure SQL
- AKS (Kubernetes), Azure Functions
- Azure Cosmos DB, Azure Cache for Redis

**מדדי תמחור:**
- מכונות Azure ב-Israel Central יקרות יותר ב-5%-12% בדרך כלל מ-West Europe
- Azure Blob Storage במחיר תחרותי עם S3 ו-Cloud Storage
- Azure מציעה תמחור הטבת היברידית: הבא רישיונות Windows/SQL Server שלך לחיסכון של עד 40%

**יתרונות Azure:**
- אינטגרציה חזקה עם אקוסיסטם Microsoft הארגוני (Active Directory, Office 365, Teams)
- Azure Government Cloud לחוזים ממשלתיים ישראליים
- תמחור Dev/Test: הנחות משמעותיות לסביבות שאינן ייצור
- Azure Reserved Instances: שנה (עד 40% הנחה) או 3 שנים (עד 65% הנחה)

**כתובת תמחור:** `https://azure.microsoft.com/en-us/pricing/calculator/`

### שלב 5: השוואת Oracle Cloud Infrastructure (il-jerusalem-1)

Oracle השיקה את אזור ישראל המרכזי שלה `il-jerusalem-1` ביולי 2021 במרכז נתונים תת-קרקעי ממוגן מתחת לאזור התעשייה הר חוצבים בירושלים (50 מטר מתחת לפני הקרקע, ארבע קומות מתחת לבניין בן 17 קומות). זה היה אזור ה-hyperscaler הראשון בישראל (לפני GCP me-west1, AWS il-central-1 ו-Azure Israel Central) והוא ממוקם לעומסי עבודה הדורשים אבטחה וזמינות גבוהה.

**שירותים זמינים ב-il-jerusalem-1:**
- Compute (VM ו-bare-metal), Block Volumes, Object Storage
- Oracle Autonomous Database (המוצר הדגל של OCI), MySQL HeatWave, PostgreSQL
- Container Engine for Kubernetes (OKE), Functions (serverless)
- Oracle Fusion Cloud Applications

**עמדת תמחור:**
- Oracle מפרסמת מחירון גלובלי אחד. שירותי OCI מתומחרים זהה בין כל האזורים כולל il-jerusalem-1 (תמיד אמתו במחירון הרשמי לפני שמצטטים)
- Egress: 10 TB/חודש ראשונים החוצה כלולים בחינם בכל אזורי OCI. זו אחת ממדיניות ה-egress האגרסיביות יותר בין ה-hyperscalers
- מחירון: `https://www.oracle.com/il-en/cloud/price-list/`
- מחשבון עלויות: `https://www.oracle.com/il-en/cloud/costestimator.html`

**מתי לשקול OCI il-jerusalem-1:**
- עומסי עבודה סביב Oracle Database / Exadata בהם הרישוי כבר מטה את הכף ל-OCI
- פריסות עם דרישות אבטחה פיזית גבוהה או תשתית קריטית, בהן המיגון הפיזי של הבונקר התת-קרקעי הוא דרישה אמיתית
- עומסי egress צפויים וכבדים בהם 10 TB/חודש החינמיים זזים את המחט
- ממשלה ותעשייה מפוקחת שצריכות אזור ישראלי אבל מעדיפות ספק מחוץ לחוזה Project Nimbus של AWS ו-Google

**מגבלות:**
- אקוסיסטם צד-שלישי וקטלוג שירותים מנוהלים קטנים יותר מ-AWS/Azure/GCP
- פחות תוכניות קרדיטים לסטארטאפים ופחות שותפויות עם מאיצים ישראליים
- חלק משירותי OCI החדשים (במיוחד יכולות generative-AI) מושקים תחילה בארה"ב או באירופה

### שלב 5ב: הערכת ספקי ענן ישראליים

למקרי שימוש ספציפיים, ספקי ענן ישראליים עשויים להציע יתרונות:

**Kamatera (`https://www.kamatera.com`):**
- חברה ישראלית עם חמישה מרכזי נתונים בישראל (פתח תקווה, חיפה ועוד)
- תמחור תחרותי: החל מכ-4 דולר/חודש ל-VPS בסיסי (1 vCPU, 1GB RAM). אמתו את המחיר העדכני בדף Kamatera לפני ציטוט
- מודל pay-as-you-go: תשלום רק על מה שמשתמשים, חיוב שעתי או חודשי, ללא חיובי תעבורה מפתיעים
- 30 יום ניסיון חינם
- מתאים ל: פרויקטים קטנים, סביבות פיתוח, אפליקציות לשוק הישראלי
- מגבלות: קטלוג שירותים קטן יותר מ-hyperscalers, ללא Kubernetes מנוהל
- חיוב בשקלים פורסם היסטורית. אם המשתמש דורש חשבונית שקלית מובטחת, אמתו ישירות מול מכירות Kamatera כי מדיניות מטבע החיוב השתנתה בין מפיצים אזוריים

**הערה:** HostIL (`hostil.co.il`) הוא ספק אחסון ישראלי שמופיע במדריכים שונים, אך לא ניתן היה לאמת באופן עצמאי את הסטטוס התפעולי הנוכחי שלו. בדקו את האתר ישירות לפני שמסתמכים עליו.

**Akamai / Linode**: ל-Akamai יש משרד בתל אביב אבל נכון למאי 2026 אין מרכז נתונים של Linode Connected Cloud בישראל. משתמשים ישראליים פונים בדרך כלל לאזורי Linode בפרנקפורט או אמסטרדם, מה שמוסיף 45-65 מ"ש השהיה. אל תמליצו על Linode כאופציה "ישראלית".

**DigitalOcean, Vultr, Hetzner, OVH**: אף אחד לא מפעיל מרכז נתונים בישראל. משתמשים ישראליים על הפלטפורמות האלה משתמשים בדרך כלל בפרנקפורט (Hetzner, DigitalOcean), שטרסבורג (OVH) או אמסטרדם, עם 45-65 מ"ש השהיה לתל אביב. Hetzner היא אופציית ה-EU הזולה ביותר לעומסי עבודה שאינם דורשים תושבות ישראלית (שרתי ענן החל מכ-4.59 יורו/חודש).

### שלב 6: השוואת תושבות נתונים וציות

שיקולי הגנת נתונים ישראליים:

**תיקון 13 לחוק הגנת הפרטיות (בתוקף מ-14 באוגוסט 2025):**
- קובע כללים מחייבים להעברת מידע אישי לחו"ל. מידע אישי של תושבי ישראל ניתן להעברה לחו"ל רק אם מדינת היעד מבטיחה הגנה "הולמת" (או חלה אחת מחריגות סעיף 2 אחרות)
- רשות הגנת הפרטיות מנהלת רשימת מדינות מאושרות. האיחוד האירופי, בריטניה וקבוצה קטנה של מדינות נוספות עם רגולציה תואמת-GDPR נמצאות ברשימה
- ב-13 באפריל 2026 פרסמה הרשות מסמך עמדה על פרשנות סעיף 2(4) לתקנות העברת מידע למאגרים בחו"ל, שהדק את התנאים המעשיים למסלול "אחריות בעל השליטה". יש להתייחס לכל ארכיטקטורה חוצת-גבולות כהחלטה מתועדת ולא כברירת מחדל
- מינוי ממונה הגנת פרטיות (DPO) נדרש כעת עבור הרבה בעלי שליטה ומחזיקי מאגרים גדולים
- חובת הודעה על אירועי אבטחה מחייבת ולוחות הזמנים הדוקים יותר מהמשטר הקודם

**שכבות סקטוריאליות שעדיין חלות:**
- מוסדות פיננסיים תחת פיקוח בנק ישראל בדרך כלל נדרשים לאחסון בישראל (הוראת ניהול בנקאי תקין 357 על מחשוב ענן ממשיכה להיות מסגרת הענן הפעילה לישויות מפוקחות)
- מידע בריאותי (קופות חולים, בתי חולים) כפוף לדרישות מחמירות של מקומיות
- מכרזים ממשלתיים דורשים לעיתים קרובות מרכזי נתונים ישראליים. עומסים מסווגים נתונים דרך פרויקט נימבוס (ראו שלב 6ב)
- PCI DSS עדיין חל בנפרד עבור מידע כרטיסי אשראי

**השוואת תושבות נתונים:**

| ספק | מרכז נתונים ישראלי | ריבונות נתונים | אישורי ציות |
|-----|-------------------|---------------|-------------|
| AWS il-central-1 | כן (תל אביב) | שליטת AWS | ISO 27001, SOC 2, PCI DSS |
| GCP me-west1 | כן (תל אביב) | שליטת Google | ISO 27001, SOC 2, PCI DSS |
| Azure Israel Central | כן (ישראל) | שליטת Microsoft | ISO 27001, SOC 2, PCI DSS, IL Gov |
| Oracle il-jerusalem-1 | כן (ירושלים, תת-קרקעי) | שליטת Oracle | ISO 27001, SOC 2, PCI DSS |
| Kamatera | כן (פתח תקווה, חיפה ועוד) | חברה ישראלית | ISO 27001 |
| HostIL (אמתו זמינות) | כן (ישראל) | חברה ישראלית | בסיסי |

**המלצה לפי רמת ציות:**
- **ציות גבוה** (פיננסים, בריאות, ממשל אזרחי): Azure Israel Central או AWS il-central-1 עם הסכם עיבוד נתונים מתאים, או Oracle il-jerusalem-1 אם Oracle Database כבר ה-system of record
- **ציות סטנדרטי** (SaaS, מסחר אלקטרוני המשרת משתמשים ישראליים): כל hyperscaler עם אזור ישראלי
- **ציות נמוך** (פרויקטים אישיים, כלים פנימיים, מידע לא ישראלי): שקלו eu-west-1 (AWS), europe-west4 (GCP אמסטרדם) או Hetzner Falkenstein/Nuremberg לחיסכון בעלויות. עשו את הבחירה החוצת-גבולות מפורשת, לא במקרה

### שלב 6ב: פרויקט נימבוס (ענן ריבוני ממשלתי)

פרויקט נימבוס הוא חוזה הענן הממשלתי הישראלי בשווי 1.2 מיליארד דולר שנחתם עם AWS ו-Google ב-2021. זה **לא שירות ציבורי לכל לקוח**. זהו שוכר ריבוני נגיש למשרדי ממשלה, גופי ביטחון ושותפים מאושרים. אל תמליצו עליו לסטארטאפים כלליים או ללקוחות מסחריים.

**מתי נימבוס רלוונטי:**
- המשתמש הוא משרד ממשלתי, ספק מגזר ביטחוני, או קבלן המבצע עומס מסווג שחייב לרוץ בתוך שוכר נימבוס
- המשתמש מעריך עם איזה hyperscaler להתיישר לקראת מכרזים ממשלתיים עתידיים (ל-AWS ול-Google יש את תשתית נימבוס. Azure ו-Oracle מתחרים בערוצי ממשל נפרדים)

**מה כדאי לדעת:**
- נימבוס משתמש בתשתית ייעודית בתוך ישראל תחת תנאי ריבונות חוזיים השונים מאזורי AWS/GCP הציבוריים
- דיווחים פומביים (סוף 2025 עד תחילת 2026) מתארים "מנגנון קריצה" המחייב את AWS ו-Google לעדכן את משרד האוצר הישראלי בקודים מוצפנים אם בית משפט זר ידרוש מידע מנימבוס. זה לא בקרה טכנית, זה תנאי חוזי, והוא ממשיך לעורר בחינה ביקורתית בקרב חוקרים ועובדים בשתי החברות

### שלב 9: חישוב עלות בעלות כוללת

**תוכניות הנחה במחויבות (תמיד אמת מול דפי התמחור הרשמיים, מאחר שהתנאים משתנים):**
- **AWS Savings Plans** (Compute ו-EC2 Instance Savings Plans): עד כ-66% הנחה ממחיר on-demand במחויבות לשנה או לשלוש שנים. גמישות יותר מ-Reserved Instances הקלאסיים, מאחר שהן חלות על מספר משפחות מכונות ואזורים (במקרה של Compute Savings Plans). RIs עדיין זמינים ל-RDS, ElastiCache, Redshift ו-OpenSearch.
- **GCP Committed Use Discounts (CUDs)**: CUDs מבוססי הוצאה ו-CUDs מבוססי משאבים ל-Compute Engine ולשירותים נוספים, וגם **Flexible CUDs** שחלים על מספר משפחות מכונות ואזורים. הנחות Sustained Use ממשיכות לחול אוטומטית על שימוש on-demand.
- **Azure Reservations** (שנה / שלוש שנים) ל-VMs, SQL DB, Cosmos DB, Cache for Redis וכו', ובנוסף **Azure Savings Plan for Compute** שגמיש יותר על פני אזורים וסדרות מכונות.

בנה השוואת עלויות מקיפה הכוללת:

1. **עלויות מחשוב**: תמחור on-demand מול reserved/committed מול savings plan מול spot/preemptible
2. **עלויות אחסון**: אחסון אובייקטים + אחסון בלוקים + אחסון מסד נתונים
3. **עלויות רשת**: העברת נתונים החוצה (egress), העברה בין-אזורית, עלויות CDN
4. **עלויות שירותים מנוהלים**: מסדי נתונים, מטמונים, תורי הודעות, ניטור
5. **עלויות תמיכה**: Basic (חינם), Developer (29 דולר/חודש), Business (מ-100 דולר/חודש), Enterprise
6. **השפעת מטבע**: תנודות תמחור בדולר עבור חברות עם תקציב בשקלים. AWS, GCP ו-Oracle מחייבים בדולרים; Azure מציעה אפשרויות חיוב בשקלים דרך הסכמי enterprise ישראליים. ממוצע USD/ILS ב-2026 נע בערך בין 3.10 ל-3.20 (לעומת כ-3.70 ב-2024), ושער הספוט נע בערך בין 2.93 ל-3.19. תקציב ענן שקלי שהוגדר לפי שער 2024 כעת נדיב יותר מדי בכ-15%-20% במונחי דולרים. זה טוב לקונים אבל קל לפספס. השתמשו בממוצע נע ל-3 חודשים (בנק ישראל מפרסם שערים יציגים יומיים) ותקצבו מחדש כל רבעון.
7. **עלויות נסתרות**: חיובי NAT Gateway (AWS), שכבת רשת פרימיום (GCP), רישום דיאגנוסטי (Azure)

### שלב 10: הצגת המלצות

מבנה ההמלצה הסופית:

1. **טבלת סיכום**: השוואת עלויות זה לצד זה לדרישות הספציפיות של המשתמש
2. **השוואה מותאמת לקרדיטים**: אם למשתמש יש קרדיטים אצל ספק כלשהו, הצג שורה או טבלה נפרדת עם העלות החודשית בפועל אחרי ניצול הקרדיטים. כמו כן, ציין באופן יזום מבצעים דומים אצל ספקים אחרים שהמשתמש אולי לא מודע אליהם.
3. **המלצה ראשית**: הספק הטוב ביותר למקרה השימוש של המשתמש עם הצדקה. אם קרדיטים מטים את הכף בצורה משמעותית לטובת ספק מסוים בטווח הקצר, ציין זאת בבירור אבל גם המלץ על הבחירה הטובה ביותר לטווח הארוך אחרי שהקרדיטים נגמרים.
4. **המלצה חלופית**: האפשרות השנייה בטובה עם הסבר על הפשרות
5. **טיפים לאופטימיזציית עלויות**: פעולות ספציפיות להפחתת עלויות (reserved instances, שימוש ב-spot, התאמת גודל)
6. **שיקולי מיגרציה**: אם המשתמש כבר על ספק, הער את מאמץ המיגרציה וחששות נעילה
7. **צעדים הבאים**: קישורים לשכבה חינמית / הצעות ניסיון להערכה מעשית

### דוגמה 1: סטארטאפ מעריך ענן למוצר SaaS חדש

המשתמש אומר: "אנחנו סטארטאפ ישראלי בשלב seed שבונה מוצר SaaS עסקי. אנחנו מצפים ל-1,000 משתמשים בשנה הראשונה, בעיקר חברות ישראליות. המחסנית שלנו היא Node.js + PostgreSQL + Redis. התקציב כ-5,000 שקלים/חודש."

פעולות:
1. הער צרכי תשתית: 2 שרתי אפליקציה (2 vCPU, 4GB כל אחד), PostgreSQL מנוהל (מקביל ל-db.t3.medium), Redis מנוהל (מקביל ל-cache.t3.small), 50GB אחסון S3
2. חשב עלויות ב-AWS il-central-1, GCP me-west1 ו-Azure Israel Central
3. בדוק זמינות קרדיטים לסטארטאפים: המלץ להגיש ל-AWS Activate, GCP for Startups ו-Microsoft Founders Hub
4. שקלל תושבות נתונים: SaaS עסקי לחברות ישראליות עשוי ליהנות מאחסון מקומי לשיחות מכירה
5. שקול Kamatera כאפשרות חסכונית לסביבות פיתוח/staging

תוצאה: המלץ על GCP me-west1 כעיקרי (הנחות שימוש מתמשך + תמחור תחרותי + Cloud Run למיקרו-שירותים) עם AWS il-central-1 כחלופה. הדגש שקרדיטים לסטארטאפים משני הספקים יכולים לכסות 12-18 חודשים של אחסון. הצע Kamatera לסביבת staging בכ-100 שקלים/חודש. עלות ייצור משוערת: 1,500-2,500 שקלים/חודש לפני קרדיטים.

### דוגמה 2: ארגון מהגר מתשתית מקומית לענן

המשתמש אומר: "אנחנו חברת שירותים פיננסיים בתל אביב עם 50 שרתים מקומיים. אנחנו צריכים לעבור לענן עם תושבות נתונים ישראלית. העומסים שלנו כוללים עיבוד עסקאות, מסדי נתוני דיווח ופורטל לקוחות."

פעולות:
1. מפה תשתית נוכחית למקבילות ענן (50 שרתים עם מפרטים שונים)
2. זהה דרישות ציות: תקנות בנק ישראל, תושבות נתונים PPA, PCI DSS לעיבוד תשלומים
3. השווה AWS il-central-1 מול Azure Israel Central (לשניהם ציות שירותים פיננסיים)
4. חשב תמחור reserved instances לעומסים צפויים (אפשרויות שנה ו-3 שנים)
5. כלול עלויות מיגרציה: כלי AWS Migration Hub או Azure Migrate, הקמת רשת, בדיקות

תוצאה: המלץ על Azure Israel Central כעיקרי בזכות הטבות רישוי היברידי (הבא רישיונות Windows/SQL Server קיימים) ועמדת ציות חזקה לשירותים פיננסיים. AWS il-central-1 כחלופה אם לצוות יש יותר מומחיות AWS. עלות חודשית משוערת: 15,000-25,000 דולר/חודש עם reserved instances ל-3 שנים. הדגש את אפשרות Azure Government Cloud למכרזים ממשלתיים עתידיים.

### דוגמה 3: מפתח בוחר אחסון לפרויקט צדדי

המשתמש אומר: "אני בונה פרויקט אישי, כלי NLP לעברית. אני צריך שרת קטן עם גישת GPU להסקה, ומסד נתונים. תקציב מינימלי."

פעולות:
1. זהה צרכי GPU: הסקה בלבד דורשת GPU קטן יותר (T4 או מקביל)
2. השווה תמחור GPU: AWS g4dn.xlarge, GCP n1-standard-4 + T4, Azure NC4as_T4_v3
3. שקול מכונות spot/preemptible לחיסכון של 60%-80% על מחשוב GPU
4. בדוק אם מכונות GPU זמינות באזורים ישראליים (זמינות מוגבלת)
5. הער חלופות: מכונות GPU של Kamatera, או Lambda Labs / RunPod להסקה בלבד

תוצאה: לפרויקט צד של הסקה בלבד, הדרך הזולה ב-2026 היא ספק GPU ייעודי (Lambda Labs, RunPod או Vast.ai) שמריץ מכונת L4 או T4 בכ-$0.40-$1.00/שעה, בשילוב עם PostgreSQL מנוהל קטן בכל מקום. GCP me-west1 Cloud SQL micro, AWS RDS db.t4g.micro ב-il-central-1, או אפילו VM של Kamatera למסד נתונים. אם הפרויקט צריך תושבות ישראלית או השהיה מתחת ל-5 מ"ש, השתמשו ב-GCP me-west1 עם מכונת GPU preemptible/spot ועם Cloud SQL micro. עלות משוערת במסלול ספק ייעודי: כ-150-350 שקלים/חודש בשער הנוכחי של כ-3.10 ILS/USD. לסטארטאפ AI ישראלי עם עומסי אימון אמיתיים (לא רק הסקה), המליצו להגיש בקשה לתוכנית תל"ם של רשות החדשנות לגישה מסובסדת ל-Nvidia B200 דרך Nebius לפני שמתחייבים להזמנת GPU ב-hyperscaler. לעלות מינימלית עוד יותר, הציעו הרצת הסקה על CPU עם מודלים מקוונטזים אם סובלנות ההשהיה מאפשרת.

## מלכודות נפוצות

- **עיוורון קרדיטים**: כשמשתמש מציין שיש לו קרדיטים או מבצעים אצל ספק מסוים, סוכנים נוטים להמליץ מיד על אותו ספק בלי לבדוק אם ספקים אחרים מציעים עסקאות דומות או טובות יותר. תמיד יש להשוות תוכניות קרדיטים בין כל הספקים לפני שממליצים. למשל, משתמש עם $100 קרדיטים ב-AWS אולי לא יודע ש-GCP מציעה $300 בקרדיטי ניסיון, ש-Azure Founders Hub מציע עד $150,000 לסטארטאפים מגובי VC, או שתוכנית תל"ם של רשות החדשנות מציעה גישה מסובסדת ל-B200 לאימון AI.
- תמחור אזור ישראל ב-AWS (il-central-1) שונה מ-eu-west-1 ומאזורים אחרים. סוכנים עלולים להשתמש בתמחור גלובלי שלא משקף את הפרמיה של האזור הישראלי.
- **Oracle il-jerusalem-1 נמצא בישראל גם אם לפעמים מתעלמים מ-OCI.** אל תגידו למשתמש "אין אזור ישראלי של Oracle". יש אזור (בבונקר תת-קרקעי בירושלים) מאז יולי 2021.
- **אל תבלבלו בין AWS me-south-1 (בחריין) לאזור הישראלי.** AWS השיקה את il-central-1 בתל אביב באוגוסט 2023. me-south-1 בבחריין כבר לא ה-fallback הקרוב ביותר למשתמשים ישראליים ואין להמליץ עליו בתור כזה.
- **ל-Akamai/Linode אין מרכז נתונים ישראלי.** ל-Akamai יש משרד בתל אביב אבל ל-Linode Connected Cloud אין מיקום בישראל. פרנקפורט היא ה-fallback הטיפוסי (45-65 מ"ש השהיה).
- עלויות ענן ישראליות צריכות להיות מחושבות בשקלים כולל 18% מע"מ ל-B2C, אך ללא מע"מ ל-B2B עם חשבונית מס תקינה. סוכנים עלולים לשכוח להוסיף מע"מ להשוואות מול צרכן.
- **העברה חוצת-גבולות דורשת בסיס מתועד תחת תיקון 13 לחוק הגנת הפרטיות** (בתוקף מ-14 באוגוסט 2025). אל תעברו בשתיקה לברירת מחדל של eu-west-1 או europe-west4 ל-SaaS ישראלי שמחזיק מידע אישי. הבחירה צריכה להתאים לאחת מחריגות סעיף 2 (רשימת מדינות מאושרות, מסלול אחריות בעל השליטה תחת הנחיית הרשות מאפריל 2026 וכו'). דרישות אחסון מידע בישראל למכרזים ממשלתיים עשויות לחייב אחסון בתוך ישראל. סוכנים עלולים להמליץ על אזורים בינלאומיים זולים יותר שמפרים דרישות אלה.
- **דריפט מטבע**: USD/ILS ירד מכ-3.70 ב-2024 לכ-3.10 ב-2026. תקציב שקלי שהוגדר בשער הישן עכשיו נדיב מדי בכ-15%-20% במונחי דולרים. כווננו את התקציב לפי השער הנוכחי במקום למחזר המרות ישנות.
- **פרויקט נימבוס הוא לא אופציה לסטארטאפים.** אל תמליצו על נימבוס ללקוחות מסחריים. זה חוזה ענן ריבוני ממשלתי על AWS ו-Google, לא מוצר ענן ציבורי.
- תקופות שבת וחגים עלולות להשפיע על תמחור וזמינות spot instances באזור ישראל בצורה שונה מאזורים אחרים בגלל ביקוש מקומי מופחת.

## פתרון בעיות

### שגיאה: "שירות לא זמין באזור הישראלי"

סיבה: לא כל שירותי הענן זמינים בכל אזור. אזורים ישראליים (il-central-1, me-west1, Israel Central) הושקו לאחרונה ועשויים לחסר שירותים חדשים או מתמחים.

פתרון:
1. בדוק את דף זמינות השירותים האזורי של הספק:
   - AWS: `https://aws.amazon.com/about-aws/global-infrastructure/regional-product-services/`
   - GCP: `https://cloud.google.com/about/locations`
   - Azure: `https://azure.microsoft.com/en-us/explore/global-infrastructure/products-by-region/`
2. השתמש בארכיטקטורת רב-אזורית: שמור נתונים באזור הישראלי, הרץ שירותים לא זמינים באזור האירופי הקרוב ביותר
3. לשירותי serverless (Lambda, Cloud Functions), בדוק אם אפשר להפעילם בין-אזורית עם השהיה מקובלת
4. צור קשר עם הצוות הישראלי של הספק למפת דרכים של זמינות שירותים

### שגיאה: "עלויות גבוהות משמעותית מהאומדן"

סיבה: גורמי עלות בלתי צפויים נפוצים כוללים חיובי העברת נתונים, עמלות NAT Gateway (AWS), רשת פרימיום (GCP) ורישום דיאגנוסטי (Azure).

פתרון:
1. הפעל התראות עלות ותקציבים במסוף החיוב של ספק הענן
2. סקור את פירוט החיוב לפי שירות וזהה את מניעי העלות המובילים
3. גורמים נפוצים ותיקונים:
   - **NAT Gateway (AWS)**: מחייב 0.045 דולר/GB מעובד. שקול VPC endpoints עבור S3/DynamoDB כדי להימנע מ-NAT
   - **העברת נתונים**: חיובי egress מצטברים. השתמש ב-CloudFront/CDN לנכסים סטטיים, דחוס תגובות API
   - **משאבים לא פעילים**: בדוק מכונות רצות, אמצעי EBS לא מחוברים, כתובות Elastic IP שאינן בשימוש
   - **מסדי נתונים בגודל עודף**: מכונות RDS/Cloud SQL לעיתים רצות בניצולת 10%-20%. התאם גודל או השתמש בגרסאות serverless
4. השתמש בכלי FinOps: AWS Cost Explorer, GCP Recommender, Azure Advisor להצעות אופטימיזציה

### שגיאה: "חיוב בשקלים לא זמין"

סיבה: hyperscalers מרכזיים (AWS, GCP) מחייבים בדולרים. תנודות מטבע מוסיפות אי-ודאות תקציבית לחברות עם תקציב בשקלים.

פתרון:
1. Azure מציעה כמה אפשרויות חיוב בשקלים להסכמי enterprise ישראליים. צור קשר עם מכירות Microsoft ישראל.
2. Kamatera מחייבת באופן מקורי בשקלים
3. לספקים שמחייבים בדולרים, השתמש באסטרטגיות אלה:
   - הגדר תקציבים עם חיץ מטבעי של 10%
   - רכוש reserved instances בתקופות שער חליפין נוח
   - השתמש בחשבונות מט"ח עסקיים כדי להפחית עמלות המרה
   - שקול חוזים עתידיים עם הבנק שלך עבור הוצאות מחויבות גדולות
4. עקוב אחר שער חליפין דולר/שקל והתאם תקציבים כל רבעון
