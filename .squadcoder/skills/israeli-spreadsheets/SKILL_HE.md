### שלב 6: Google Sheets (RTL ושקלים)

משתמשים רבים עובדים ב-Google Sheets ולא ב-Excel. לגיליונות בעברית:

- מגדירים את כיוון הגיליון דרך **Sheet > Right-to-left** (או כפתור ה-RTL בסרגל הכלים). זה הופך את סדר העמודות כך שעברית נקראת באופן טבעי.
- מגדירים את ה-locale של הגיליון דרך **File > Settings > Locale > Israel** כדי שתאריכים יפורשו כ-DD/MM/YYYY ומטבע ברירת המחדל יהיה שקל.
- מעצבים תאי מטבע דרך **Format > Number > Custom number format** עם `#,##0.00 ₪` (או הפורמט המובנה לשקל מתוך ה-locale).
- לשערי חליפין חיים משתמשים ב-`GOOGLEFINANCE`, למשל `=GOOGLEFINANCE("CURRENCY:USDILS")` מחזיר את שער הדולר לשקל. מכפילים סכום בדולרים בערך הזה כדי להמיר לשקלים.
- ל-Google Sheets אין API בסגנון `openpyxl`, בונים תבניות ידנית או עם Apps Script. קבועי המס ולוגיקת המע"מ של הסקיל הזה תקפים באותה מידה.
- עזרי Gemini: התפריט **Insert > Smart fill** ופונקציית `=AI("הנחיה", טווח)` בתוך תא (זמינה לרוב חבילות Workspace במהלך 2026) יודעים לקטלג טקסט בעברית, לסכם עמודה או לנסח נוסחה מתיאור במילים. תתייחסו לפלט של `=AI()` כטיוטה: תלוש משכורת או חשבונית ישראלית עדיין דורשים את חשבונאות המע"מ ומדרגות המס המדויקות, לא ניחוש של מודל שפה.

### שלב 6.5: נוסחאות ישראליות שימושיות

נוסחאות לשימוש חוזר שמתאימות לקבועים שלמעלה. עובדות באופן זהה ב-Excel וב-Google Sheets אלא אם צוין אחרת.

**מע"מ (18%) בתא בודד:**

```
נטו לברוטו:                  =A1*1.18
ברוטו לנטו:                   =A1/1.18
סכום המע"מ במחיר כולל מע"מ:  =A1-A1/1.18    (או =A1*0.18/1.18)
סכום המע"מ במחיר לפני מע"מ:  =A1*0.18
```

**מחרוזת פורמט שקלי (פורמט מותאם של Excel, הסימן אחרי המספר כמקובל בישראל):**

```
#,##0.00 [$₪-he-IL]
```

הצורה הקצרה `#,##0.00 "₪"` גם עובדת; הצורה עם `[$₪-he-IL]` נושאת תג locale עברי-ישראל ולכן הסימן שורד שמירה חוזרת ב-Excel שעובד ב-locale אחר.

**חישוב מס הכנסה פרוגרסיבי (מדרגות 2026, הכנסה שנתית ב-A1, תושבי ישראל) -- Google Sheets / Excel 365:**

```
=LET(inc, A1, credit, 2.25*2904,
  MAX(0,
    MIN(inc,84120)*0.10
    + MAX(0, MIN(inc,120720)-84120)*0.14
    + MAX(0, MIN(inc,228000)-120720)*0.20
    + MAX(0, MIN(inc,301200)-228000)*0.31
    + MAX(0, MIN(inc,560280)-301200)*0.35
    + MAX(0, MIN(inc,721560)-560280)*0.47
    + MAX(0, inc-721560)*0.50
    - credit))
```

`LET` שומר את הנוסחה קריאה ומחשב את `A1` פעם אחת במקום שבע פעמים. בגרסאות Excel ישנות פותחים את `LET` לתאים נפרדים.

**ביטוח לאומי + מס בריאות (חלק העובד, שכר חודשי ב-A1):**

```
ביטוח לאומי:  =MIN(A1,7122)*0.004 + MAX(0, MIN(A1,49030)-7122)*0.07
מס בריאות:    =MIN(A1,7122)*0.031 + MAX(0, MIN(A1,49030)-7122)*0.05
```

הכנסה מעל תקרת ההכנסה המבוטחת של 49,030 ש"ח/חודש אינה מבוטחת, ולכן שתי הנוסחאות חסומות בתקרה הזו.

**יום בשבוע בעברית מתאריך לועזי ב-A1 (תלוי locale):**

```
=TEXT(A1,"dddd")
```

מחזיר "ראשון", "שני" וכו' כאשר ה-locale של חוברת העבודה הוא he-IL. ב-locale אחר משתמשים ב-`=CHOOSE(WEEKDAY(A1,1),"ראשון","שני","שלישי","רביעי","חמישי","שישי","שבת")`.

**חגים ישראליים דרך Hebcal:** ל-Hebcal אין פונקציית גיליון ישירה. שתי דרכים:
- לייצא CSV של טווח שנתי מ-`hebcal.com` (חגים, מועדים, פרשת השבוע) ולעשות `VLOOKUP` כנגד עמודת התאריך הלועזי, או
- ב-Google Sheets להפעיל קריאת REST ל-Hebcal דרך Apps Script (`UrlFetchApp.fetch('https://www.hebcal.com/hebcal?cfg=json&year=2026&maj=on&geo=geoname&geonameid=293397')`) ולכתוב את התוצאה לגיליון מוסתר. שרת ה-MCP `hebcal` (ראו שלב 7) הוא הדרך הנקייה ביותר כשסוכן בונה את הקובץ.

### שלב 7: שרתי MCP מומלצים

בעת בניית גיליונות פיננסיים, שרתי ה-MCP הבאים מהדירקטוריה מספקים נתונים חיים כדי שהמספרים יישארו עדכניים:

- **boi-exchange**: שערי חליפין של בנק ישראל, שימושי לכל גיליון שממיר מטבע זר לשקלים או עוקב אחרי השער היציג.
- **hebcal**: תאריכים עבריים ולוח השנה היהודי, שימושי כשגיליון צריך תאריכים עבריים (למשל חשבונית עם תאריך עברי) או צריך לדלג על שבת וחגים.

**טיפול בתאריך עברי:** openpyxl כותב רק תאריכים לועזיים. כדי להציג תאריך עברי (למשל "כ״ה אדר תשפ״ו") לצד הלועזי, מחשבים אותו דרך שרת ה-MCP `hebcal` או ספרייה כמו `pyluach`, ואז כותבים אותו כמחרוזת טקסט בתא נפרד. אין לנסות לעצב תא תאריך לועזי כעברי, ההמרה חייבת לקרות לפני שהערך מגיע לגיליון.

## מלכודות נפוצות

- טקסט בעברית בגיליונות אלקטרוניים דורש יישור תא RTL. יישור LTR ברירת מחדל גורם לעברית להיראות לא נכון, עם סימני פיסוק ומספרים בצד הלא-נכון.
- פורמט תאריך ישראלי בגיליונות אלקטרוניים הוא DD/MM/YYYY, לא MM/DD/YYYY. Excel ו-Google Sheets עלולים לפענח "01/03/2026" כ-1 בינואר (אמריקאי) במקום 1 במרץ (ישראלי). תמיד הגדירו locale ל-Hebrew (Israel).
- עיצוב מטבע שקלי מציב את הסימן אחרי המספר, לא לפניו (סגנון אמריקאי). גם `₪` וגם הקיצור `ש"ח` מקובלים בישראל. הסקיל הזה והסקריפט המצורף מתקננים על `#,##0.00 ₪` (הסימן אחרי המספר). בחרו מוסכמה אחת והשתמשו בה בעקביות בכל חוברת העבודה, אל תערבבו `₪` ו-`ש"ח` באותו גיליון.
- חישובי מס ישראליים בגיליונות חייבים להתחשב במע"מ בשיעור 18%. סוכנים עלולים לקודד שיעורי מע"מ ישנים (17%) מנתוני אימון לפני 2025.
- מדרגות מס ההכנסה ל-2026 שונות מאלו של 2025. תיקון 288 הרחיב את מדרגות 20% ו-31% החל מ-1.1.2026 (20% עד 228,000 ש"ח לשנה, 31% עד 301,200, ה-35% מתחיל ב-301,201). תבניות תלוש שהועתקו ממקורות של 2024-2025 ינפחו את המס של עובדים במעמד הביניים.
- ה-SORT של Excel ומיון לפי כותרת עמודה מסתדרים עם האות הראשונה בלבד בכמה בילדים; משם והלאה החזרה לסדר Unicode, שלא מתלכד עם הסדר האלפביתי העברי. ברשימות שמות עבריות צפו ל-5%-10% טעויות סדר ובדקו ידנית, או מיינו ב-Google Sheets שמשתמש ב-collation שמטפל בעברית כראוי.
- תאים ממוזגים בגיליון RTL לפעמים "מתהפכים" כשהקובץ נפתח ב-Excel עם locale לא ישראלי: הטווח הממוזג מוצג בכיוון ההפוך גם כאשר `rightToLeft` מוגדר. בגיליונות RTL כדאי להימנע ממיזוג שורות; להשתמש בעמודה רחבה עם טקסט ממורכז במקום.

## פתרון בעיות

### שגיאה: "סימן השקל מופיע בצד הלא נכון של המספר"
סיבה: הגדרת Locale של Excel לא מוגדרת לעברית/ישראל
פתרון: השתמשו במחרוזת פורמט `#,##0.00 ₪` (סימן אחרי המספר) בהתאם למוסכמה הישראלית, או להגדיר את ה-locale של חוברת העבודה ל-he-IL.

### שגיאה: "כותרות עמודות בעברית מוצגות כסימני שאלה"
סיבה: חוברת העבודה לא נשמרה בקידוד UTF-8 או שהגופן לא תומך בעברית
פתרון: תוודאו שחוברת העבודה משתמשת בגופן תואם Unicode (כגון David, Heebo או Arial). בעת שימוש ב-openpyxl, מחרוזות עברית מקודדות אוטומטית ב-UTF-8.
