# חיבורי Zapier לישראל

## הוראות

### שלב 1: בחירת דפוס ה-Zap המתאים

התאם את הצורך העסקי למבנה ה-Zap הנכון.

| צורך עסקי | טריגר Zap | שרשרת פעולות | אפליקציות ישראליות |
|-----------|-----------|-------------|-------------------|
| קבלה אוטומטית אחרי תשלום | Webhook של קארדקום/טרנזילה | ניתוח תשלום -> יצירת מסמך ב-Morning -> מייל ללקוח | קארדקום, Morning |
| סנכרון חשבונית להנהלת חשבונות | Webhook מסמך חדש ב-Morning | מיפוי שדות -> יצירת רשומה ב-Zapier Tables או כלי הנהלת חשבונות -> תיוג תקופת מע"מ | Morning, Zapier Tables |
| תזכורת תשלום (לעצמאים) | טריגר מתוזמן (דו-חודשי) | שליפת חשבוניות שלא שולמו -> סינון חובות -> שליחת תזכורת בעברית | Morning, ספק מייל/SMS |
| עיבוד הזמנות e-commerce | הזמנה חדשה ב-WooCommerce/Shopify | יצירת חשבונית ב-Morning -> מייל אישור -> עדכון בורד Monday.com | Morning, Monday.com |
| אישור הזמנה בוואטסאפ | Webhook תשלום | עיצוב הודעה בעברית -> שליחה דרך Twilio WhatsApp Business API | Twilio (WhatsApp Business) |
| סיווג הוצאות | מייל (קבלה מצורפת) | ניתוח קבלה -> סיווג לפי סוג ניכוי מס -> רישום ב-Zapier Tables | Gmail, Zapier Tables |
| לכידת לידים עם CRM | הגשת טופס (Typeform, Google Forms) | חילוץ שם בעברית -> יצירת איש קשר ב-CRM -> מייל מעקב | Monday.com, HubSpot |
| איחוד סליקות ממספר שערים | Webhooks מרובים (קארדקום, טרנזילה, Grow) | נרמול סכומים -> רישום בטבלת Zapier אחידה | קארדקום, טרנזילה, Grow |

**בחירה בין Zap חד-שלבי לרב-שלבי:**
- Zap חד-שלבי (חינמי): טריגר ישירות לפעולה, למשל "תשלום חדש בקארדקום -> שורה חדשה ב-Zapier Tables". המסלול החינמי כולל Zaps ללא הגבלה אך רק שני שלבים (טריגר + פעולה אחת), 100 משימות בחודש, ופולינג כל 15 דקות.
- Zap רב-שלבי (Starter, $19.99/חודש בתשלום שנתי או $29.99/חודש, 750 משימות): שרשור פעולות עם לוגיקה, למשל "תשלום חדש -> יצירת חשבונית -> מייל -> עדכון CRM". מסלול Professional ($49/חודש שנתי, 2,000 משימות) מוסיף פילטרים מתקדמים ושלבים מותנים.
- השתמש ב-Paths (הסתעפות) כשה-Zap צריך לטפל בתרחישים שונים, למשל "אם התשלום עובר את סף רפורמת החשבוניות (10,000 ש"ח עד 31 במאי 2026, ואז 5,000 ש"ח החל מ-1 ביוני 2026), הוסף מספר הקצאה של רפורמת החשבוניות". שמור את הסף כמשתנה ב-workflow ולא כמספר קשיח, מאחר שהוא מתוכנן לרדת שוב.

**השתמש ב-Zapier Copilot** (זמין בחינם בכל המסלולים) כדי לתאר מה שאתה רוצה בשפה חופשית. Copilot מציע מבנה Zap, מוצא את האפליקציות המתאימות וממפה שדות אוטומטית. דוגמה: "כשאני מקבל תשלום בקארדקום, תיצור קבלה ב-Morning ותשלח אותה במייל ללקוח."

### שלב 2: חיבור אפליקציות ישראליות ל-Zapier

אפליקציות ישראליות מתחברות ל-Zapier בשלוש דרכים. בחר לפי מה שהאפליקציה תומכת.

| אפליקציה | שיטת חיבור | סוג אימות | הערות |
|----------|-----------|----------|-------|
| Morning (חשבונית ירוקה) | Webhooks by Zapier | מפתח API + Webhook | אין אפליקציה מובנית של Morning ב-Zapier. חיבור דרך webhooks: שימוש ב-webhook notifications של Morning כטריגרים וב-REST API שלהם (`api.greeninvoice.co.il`) דרך Webhooks by Zapier לפעולות. |
| קארדקום (Cardcom) | Webhooks by Zapier | IndicatorUrl (callback בשיטת GET) | קארדקום קורא ל-webhook שלך עם פרמטרים ב-GET על אירועי תשלום. הגדרה דרך IndicatorUrl בהגדרות המסוף. |
| טרנזילה (Tranzila) | Webhooks by Zapier | כתובת Webhook | טרנזילה V2 משתמשת ב-iframe עם hosted fields. הגדרת כתובת התראה בפאנל הסוחר לקבלת callbacks אחרי תשלום. |
| Monday.com | אינטגרציה מובנית ב-Zapier | OAuth | תמיכה מלאה. |
| iCount | Webhooks by Zapier | מפתח API | מערכת הנהלת חשבונות ישראלית עם REST API ליצירת חשבוניות, קבלות וניהול לקוחות. |
| EZcount | Webhooks by Zapier | מפתח API | פלטפורמת חשבוניות ישראלית פופולרית עם REST API. |
| Grow by Meshulam | Webhooks by Zapier | כתובת Webhook | שער תשלומים ישראלי שתומך בכרטיסי אשראי, ביט, Apple Pay, Google Pay. שולח webhooks בפורמט JSON. |
| Sumit | Webhooks by Zapier | מפתח API | פלטפורמת חשבוניות וקבלות ישראלית עם REST API. |
| ריווחית (Rivhit) | Webhooks by Zapier | מפתח API ב-header | אין אינטגרציה מובנית. webhook + קריאות API. |
| Priority ERP | Webhooks by Zapier | Basic auth | שימוש ב-OData REST API של Priority. |
| ספקי SMS (019, InforUMobile) | Webhooks by Zapier | מפתח API | שליחת SMS דרך פעולת HTTP POST. |

**הגדרת API של Morning (חשבונית ירוקה):**
1. היכנס לדשבורד של Morning באתר app.greeninvoice.co.il
2. נווט להגדרות > אינטגרציית API
3. צור מפתח API חדש (אימות מבוסס JWT)
4. ב-Zapier, השתמש ב-"Webhooks by Zapier" עם Custom Request כדי לקרוא ל-REST API של Morning בכתובת `api.greeninvoice.co.il`
5. הגדר את כותרת Authorization עם ה-JWT token

**קודי סוגי מסמכים ב-Morning** (השתמש בקודים המספריים בקריאות API):

| קוד | סוג מסמך |
|-----|----------|
| 10 | הצעת מחיר |
| 305 | חשבונית מס |
| 320 | חשבונית מס קבלה |
| 330 | חשבונית זיכוי / החזר |
| 400 | קבלה |

**הגדרת IndicatorUrl של קארדקום:**
1. ב-Zapier, צור Zap חדש עם "Webhooks by Zapier" כטריגר
2. בחר "Catch Hook" כאירוע טריגר
3. העתק את כתובת ה-webhook שנוצרה
4. בהגדרות מסוף קארדקום, הדבק את הכתובת בשדה IndicatorUrl
5. קארדקום שולח בקשת GET עם פרמטרים (לא JSON POST) בסיום תשלום
6. פרמטרים עיקריים: `InternalDealNumber` (מזהה עסקה), `Amount` (שקלים עשרוניים, למשל 150.50), `CardOwnerName`, `CardOwnerEmail`, `CardOwnerPhone`, `NumOfPayments`
7. בצע תשלום ניסיון כדי לשלוח נתוני דוגמה ל-Zapier

**הגדרת טרנזילה (API V2 מודרני):**
טרנזילה עברה ל-API V2 מבוסס iframe עם hosted fields לתאימות PCI. להגדרת webhooks אחרי תשלום:
1. הגדר את כתובת ההתראה בפאנל הסוחר של טרנזילה
2. טרנזילה שולחת אישור תשלום עם שדות: `index` (מזהה עסקה), `sum` (סכום בשקלים), `ccno` (4 ספרות אחרונות), `npay` (תשלומים)
3. הגישה הישנה מבוססת ok_page/fail_page עדיין עובדת אך מיושנת

### שלב 3: טיפול בטקסט עברי בשלבי Zap

טקסט עברי דורש טיפול מיוחד ב-Zapier כדי למנוע בעיות תצוגה וקידוד.

**ניקוי טקסט עברי עם סמני כיוון Unicode:**
פונקציית "Formatter > Text > Trim Whitespace" של Zapier מסירה רווחים רגילים אך לא סמני כיוון Unicode (U+200F RLM, U+200E LRM). לניקוי מעמיק של קלט עברי, השתמש בשלב "Code by Zapier":

```javascript
const cleaned = inputData.text.replace(/[\u200F\u200E\u200B\u200C\u200D\uFEFF]/g, '').trim();
output = [{text: cleaned}];
```

**עיצוב שמות:** שמות בעברית הם "פרטי משפחה" (ללא שם אמצעי). השתמש ב-"Formatter > Text > Titlecase" רק לשמות באנגלית. לעברית, העבר כמות שהוא.

**טקסט דו-כיווני:** בעת שרשור טקסט עברי ואנגלי (למשל "Order #12345 - הזמנה חדשה"), מקם את החלק האנגלי קודם, ואז עברית. Zapier מציג טקסט דו-כיווני ב-LTR כברירת מחדל.

**תבניות אימייל בטוחות ל-RTL:**
בשליחת אימיילים ב-HTML דרך Zapier, עטוף תוכן עברי עם כיוון מפורש:

```html
<div dir="rtl" style="text-align: right; font-family: Arial, sans-serif;">
  <p>שלום {{customer_name}},</p>
  <p>קיבלנו את התשלום שלך בסך {{amount}} ש"ח.</p>
  <p>מספר חשבונית: {{invoice_number}}</p>
</div>
```

**פורמט מטבע שקלי (ILS):**
ה-Formatter המובנה של Zapier לא תומך ב-ILS. השתמש ב-Formatter by Zapier > Numbers > Format Number:
- שתי ספרות אחרי הנקודה
- מפריד עשרוני: `.` (נקודה)
- מפריד אלפים: `,` (פסיק)
- אז צרף " ש\"ח" או " ILS" בשלב שרשור טקסט

**פורמט תאריכים לישראל:**
מסמכים ישראליים משתמשים ב-DD/MM/YYYY. השתמש ב-Formatter by Zapier > Date/Time > Format עם:
- פורמט: `DD/MM/YYYY`
- אסור להשתמש בברירת המחדל MM/DD/YYYY

### שלב 4: בניית אוטומציות מחזור חיוב ישראלי

דיווח מס בישראל עוקב אחר מחזורים מוגדרים. בנה Zaps שמתואמים עם תקופות אלו.

**Zaps לתקופת מע"מ דו-חודשית:**
עסקים ישראליים מדווחים מע"מ דו-חודשית: ינואר-פברואר, מרץ-אפריל, מאי-יוני, יולי-אוגוסט, ספטמבר-אוקטובר, נובמבר-דצמבר. הדוח נדרש עד ה-15 לחודש שאחרי התקופה (דיווח מקוון מאריך ל-19 לחודש). השתמש ב-Schedule by Zapier:
- טריגר: תזמון לחודשים ספציפיים (מרץ, מאי, יולי, ספטמבר, נובמבר, ינואר) ביום ה-10 לחודש
- פעולה: שלוף את כל החשבוניות/קבלות מהחודשיים הקודמים דרך API של Morning והכן לדיווח מע"מ
- גישה זו פשוטה יותר מטריגר חודשי עם סינון חודשים אי-זוגיים

**תזכורות מקדמות מס (מקדמות):**
עצמאים משלמים מקדמות מס (מקדמות) דו-חודשיות (חלקם חודשיות, תלוי בסיווג רשות המסים). התשלום נדרש עד ה-15 לחודש שאחרי תקופת הדיווח (דיווח מקוון מאריך ל-19). הגדר תזכורות:
- טריגר מתוזמן: חודשי ב-10 לחודש
- סינון: רק בחודשים שבהם חלות מקדמות (תלוי בלוח הזמנים האישי, בדרך כלל דו-חודשי)
- פעולה: שלח מייל תזכורת עם מועד התשלום הקרוב וסכום משוער

**טריגרים לדוח שנתי:**
- תזמון: 15 בינואר
- פעולה: הכן סיכום שנתי מ-Morning, שלח למייל רואה החשבון
- כולל: הכנסות כוללות, הוצאות כוללות, מע"מ ששולם, ניכוי מס במקור

**תאריכי מס מרכזיים לקידוד ב-Zaps:**

| תאריך | אירוע | פעולת Zap |
|--------|--------|----------|
| 15 לחודש שאחרי תקופה דו-חודשית (19 מקוון) | מועד דיווח מע"מ | שליחת תזכורת 5 ימים לפני |
| 15 לחודש שאחרי תקופת דיווח (19 מקוון) | מועד מקדמות מס | שליחת תזכורת + הערכת סכום |
| 15 בינואר | סיכום שנתי של השנה הקודמת | הכנה ושליחה לרואה חשבון |
| 30 באפריל | מועד הגשת דוח שנתי (סטנדרטי) | תזכורת הגשה |
| 31 במאי ואילך | מועד מוארך (עם ייצוג רואה חשבון) | תזכורת הגשה |

**רפורמת חשבוניות 2026:**
חשבוניות מס מעל הסף דורשות מספר הקצאה מרשות המסים. הסף יורד במהלך 2026:

| תאריך כניסה לתוקף | סף |
|--------------------|-----|
| 1 בינואר 2026 | 10,000 ש"ח |
| **1 ביוני 2026** | **5,000 ש"ח** |
| 1 בינואר 2027 | 5,000 ש"ח (מתוכנן להמשיך) |

בבניית Zaps ליצירת חשבוניות:
- הוסף שלב Filter שמשווה את סכום החשבונית למשתנה ב-workflow שמכיל את הסף הנוכחי, וסמן לקבלת מספר הקצאה כשהוא עובר את הסף. בנו את בדיקת הסף כמשתנה ב-workflow, לא כמספר קשיח, מאחר שהסף מתוכנן לרדת שוב.
- מספר ההקצאה חייב להתקבל ממערכת רשות המסים לפני הנפקת החשבונית
- Morning ופלטפורמות חשבוניות מורשות אחרות מטפלות בזה אוטומטית דרך ה-API שלהן, אך ודא שתגובת המסמך כוללת מספר הקצאה
- בתהליכים ידניים מבוססי webhook, הוסף שלב Code by Zapier שקורא ל-API של רשות המסים לפני יצירת החשבונית

### שלב 5: הגדרת חיבורים ישראליים מבוססי Webhook

מעבדי תשלומים ישראליים רבים חסרים אינטגרציה מובנית ב-Zapier. השתמש ב-webhooks כגשר.

**Webhook תשלום קארדקום כטריגר Zap:**

קארדקום משתמש במנגנון IndicatorUrl. בסיום עסקה, קארדקום שולח בקשת GET לכתובת ה-webhook שלך עם פרטי התשלום כפרמטרים. שדות עיקריים:

| שדה | תיאור | דוגמה |
|-----|--------|-------|
| `InternalDealNumber` | מזהה עסקה | `12345678` |
| `Amount` | סכום תשלום בשקלים עשרוניים | `150.50` (= 150.50 ש"ח) |
| `CardOwnerName` | שם בעל הכרטיס | `ישראל ישראלי` |
| `CardOwnerEmail` | אימייל לקוח | `israel@example.com` |
| `CardOwnerPhone` | טלפון לקוח | `0541234567` |
| `NumOfPayments` | מספר תשלומים | `3` |
| `DealResponse` | קוד תגובה (0 = הצלחה) | `0` |
| `CardNum` | 4 ספרות אחרונות | `1234` |

סכומים בקארדקום הם בשקלים עשרוניים (למשל 150.50 = 150.50 ש"ח). אין צורך בהמרה. השתמש בערך ישירות בשלב יצירת החשבונית.

**Webhook תשלום טרנזילה:**

שדות מרכזיים להתראות תשלום של טרנזילה:

| שדה | תיאור | דוגמה |
|-----|--------|-------|
| `index` | מזהה עסקה | `87654321` |
| `sum` | סכום בשקלים עשרוניים | `250.00` |
| `ccno` | 4 ספרות אחרונות | `5678` |
| `npay` | תשלומים | `1` |
| `contact` | שם לקוח | `ישראל ישראלי` |
| `email` | אימייל לקוח | `israel@example.com` |
| `phone` | טלפון לקוח | `0541234567` |

גם קארדקום וגם טרנזילה שולחים סכומים בשקלים עשרוניים. אין צורך בהמרת יחידות בשום מעבד תשלומים ישראלי.

**Webhook תשלום Grow by Meshulam:**

Grow תומך בכרטיסי אשראי, ביט, Apple Pay ו-Google Pay. שולח webhooks בפורמט JSON:

| שדה | תיאור | דוגמה |
|-----|--------|-------|
| `transaction_id` | מזהה עסקה | `GRW-123456` |
| `amount` | סכום בשקלים עשרוניים | `99.90` |
| `payment_method` | סוג תשלום | `credit_card`, `bit`, `apple_pay`, `google_pay` |
| `customer_name` | שם לקוח | `ישראל ישראלי` |
| `customer_email` | אימייל לקוח | `israel@example.com` |
| `customer_phone` | טלפון לקוח | `0541234567` |

**תשלומי ביט:** ביט היא אפליקציית התשלומים P2P הדומיננטית בישראל עם אימוץ גובר בעסקים. לקבלת תשלומי ביט והפעלת Zaps, השתמש באחד השערים הבאים:
- Grow by Meshulam (תמיכה מובנית בביט דרך דף הקופה שלהם)
- טרנזילה (אינטגרציית ביט זמינה)
- API ישיר של ביט ביזנס (דורש הסכם סוחר נפרד)

### שלב 6: שליחת הודעות וואטסאפ עסקי (מגבלות וחלופות)

**האינטגרציה המובנית של Zapier עם וואטסאפ לא יכולה לשלוח הודעות ללקוחות.** נכון לינואר 2026, אינטגרציית וואטסאפ הישירה של Zapier יכולה לשלוח הודעות רק לעצמך (בעל החשבון). היא תומכת ב-7 תבניות מוגדרות מראש באנגלית בלבד ולא יכולה לשלוח הודעות מותאמות בעברית ללקוחות.

לשליחת הודעות וואטסאפ ללקוחות מתוך Zaps, השתמש באחד הספקים הבאים:

| ספק | אינטגרציית Zapier | תמיכה בעברית | דרישת אישור |
|-----|-------------------|-------------|-------------|
| Twilio WhatsApp Business API | אפליקציה מובנית ("Twilio") | כן, דרך תבניות מאושרות | אימות Meta Business + אישור תבניות |
| WATI | אפליקציה מובנית ("WATI") | כן, דרך תבניות מאושרות | אימות Meta Business + אישור תבניות |
| Respond.io | אפליקציה מובנית ("Respond.io") | כן | אימות Meta Business |

**נקודות חשובות לגבי וואטסאפ עסקי:**
- כל הודעות וואטסאפ ללקוחות דורשות תבניות הודעה שאושרו מראש על ידי Meta
- התבניות צריכות להיות מוגשות לאישור מראש (בדרך כלל 24-48 שעות)
- תבניות בעברית נתמכות אך צריך להגיש אותן עם הטקסט בעברית
- אי אפשר לשלוח הודעות חופשיות ללקוחות, רק למלא משתנים בתבנית מאושרת
- דוגמה לתבנית מאושרת: "שלום {{1}}, קיבלנו את התשלום שלך בסך {{2}} ש\"ח. מספר אישור: {{3}}. תודה!"
- להתראות פנימיות לעצמך או לצוות שלך, אינטגרציית וואטסאפ הרגילה של Zapier עובדת

### שלב 7: שימוש ב-Zapier Tables ו-Interfaces (2026)

Zapier Tables ו-Interfaces חינמיים בכל המסלולים ב-2026 ומספקים חלופה טובה יותר ל-Google Sheets לתהליכי עבודה עסקיים רבים.

**Zapier Tables** (חלופה ל-Google Sheets לנתונים מובנים):
- בסיס נתונים מובנה בתוך Zapier, לא צריך חיבור לאפליקציה חיצונית
- תומך בסוגי שדות: טקסט, מספר, תאריך, אימייל, URL, רשימה נפתחת, תיבת סימון
- תצוגות, סינונים ורשומות מקושרות מובנים
- טריגרים זמינים: "רשומה חדשה" ו"רשומה עודכנה" יכולים להפעיל Zaps
- עדיף עבור: מאגרי לקוחות, יומני חשבוניות, רשומות תשלום, מעקב הוצאות

**מתי להשתמש ב-Tables מול Google Sheets:**

| תרחיש | Zapier Tables | Google Sheets |
|--------|---------------|---------------|
| יומן תשלומים פשוט | כן (מהיר יותר, ללא אימות) | מוגזם |
| שיתוף עם רואה חשבון | לא (רו"ח צריך גישת Google) | כן |
| רשימת לקוחות בסגנון CRM | כן (רשומות מקושרות, תצוגות) | מוגבל |
| נוסחאות מורכבות/טבלאות ציר | לא | כן |
| דוח תקופת מע"מ | שניהם עובדים | כן אם רו"ח צריך גישה ישירה |

**Zapier Interfaces** (טפסים ודשבורדים ללא קוד):
- בנה טפסי קליטת לקוח, דפי בקשת תשלום ודשבורדים ללא כתיבת קוד
- טפסים מזינים ישירות ל-Zapier Tables או מפעילים Zaps
- שימושי עבור: טופס קליטת לקוח חדש לפרילנסרים, קישורי בקשת תשלום, טפסי משוב שירות

### שלב 9: שימוש בתבניות Zap נפוצות לעסקים ישראליים

**תבנית 1: חשבונית-להנהלת-חשבונות (לעצמאים)**
1. טריגר: Webhook של Morning (מסמך חדש נוצר)
2. סינון: סוג מסמך = חשבונית מס (305) או חשבונית מס קבלה (320)
3. פעולה: יצירת רשומה ב-Zapier Tables עם עמודות: תאריך, שם לקוח, סכום (לפני מע"מ), סכום מע"מ, סה"כ, מספר מסמך
4. פעולה: אם הסכום עובר את משתנה הסף של רפורמת החשבוניות (10,000 ש"ח עד 31 במאי 2026, ואז 5,000 ש"ח החל מ-1 ביוני 2026), ודא שמספר הקצאה של רפורמת החשבוניות קיים
5. פעולה: אם הסכום מעל 25,000 ש"ח, שלח התראה ב-Slack לערוץ רואה חשבון

**תבנית 2: הזמנה-לחשבונית (לחנות אונליין)**
1. טריגר: Shopify/WooCommerce > הזמנה חדשה
2. פעולה: יצירת מסמך ב-Morning דרך Webhooks by Zapier (סוג: 320 חשבונית מס קבלה או 400 קבלה לפי העדפת העסק)
3. פעולה: שליחת אישור מייל עם פרטי הקבלה (תבנית HTML ב-RTL)
4. פעולה: עדכון בורד Monday.com עם סטטוס הזמנה

**תבנית 3: תשלום-לקבלה (קארדקום)**
1. טריגר: Webhooks by Zapier (callback מסוג GET של IndicatorUrl קארדקום)
2. סינון: `DealResponse` = 0 (תשלום מוצלח בלבד)
3. פעולה: Morning API > יצירת מסמך (סוג: 400 קבלה). שדה הסכום משתמש בערך `Amount` ישירות (כבר בשקלים עשרוניים).
4. פעולה: שליחת מייל עם קישור ל-PDF הקבלה ללקוח
5. פעולה: רישום ב-Zapier Tables להתאמה

**תבנית 4: לכידת ליד למעקב CRM**
1. טריגר: Typeform/Google Forms > תגובה חדשה
2. פעולה: Code by Zapier לניקוי טקסט עברי (הסרת סמני כיוון Unicode)
3. פעולה: יצירת איש קשר ב-CRM (Monday.com או HubSpot)
4. פעולה: שליחת מייל קבלת פנים בתבנית HTML עם RTL
5. פעולה: יצירת משימת מעקב ב-Monday.com לעוד 3 ימים

**תבנית 5: קבלת הוצאה לסיווג**
1. טריגר: Gmail > מייל חדש עם קובץ מצורף התואם "קבלה" או "חשבון"
2. פעולה: Code by Zapier לחילוץ סכום וסיווג לפי שולח
3. סינון: המשך רק אם הסכום אפשר לניתוח
4. פעולה: יצירת רשומה ב-Zapier Tables "ניכויי מס" עם עמודת קטגוריה

**תבנית 6: איחוד תשלומים ממספר שערים**
צור 3 Zaps נפרדים, כולם כותבים לאותה טבלת Zapier:
- Zap א: Webhook של IndicatorUrl קארדקום -> Zapier Tables (סכום בשקלים עשרוניים)
- Zap ב: Webhook של טרנזילה -> Zapier Tables (סכום בשקלים עשרוניים)
- Zap ג: Webhook של Grow by Meshulam -> Zapier Tables (סכום בשקלים עשרוניים)
כל שלושת המעבדים שולחים סכומים בשקלים עשרוניים. אין צורך בהמרה.

## דוגמאות

### דוגמה 1: קבלה אוטומטית לתשלומי קארדקום

המשתמש אומר: "אני רוצה ליצור קבלה אוטומטית ב-Morning כשמישהו משלם דרך קארדקום"

פעולות:
1. יצירת Zap עם Webhooks by Zapier > Catch Hook כטריגר
2. הגדרת IndicatorUrl של קארדקום לכתובת ה-webhook
3. הוספת שלב Filter: המשך רק אם `DealResponse` = 0 (תשלום מוצלח)
4. הוספת Webhooks by Zapier > Custom Request לקריאה ל-Morning API, יצירת מסמך סוג 400 (קבלה). שדה `Amount` של קארדקום הוא כבר בשקלים עשרוניים (למשל 150.50 = 150.50 ש"ח), אין צורך בהמרה.
5. מיפוי שדות: `CardOwnerName` לשם לקוח, `CardOwnerEmail` למייל לקוח, `Amount` למחיר פריט
6. אם הסכום עובר את משתנה הסף של רפורמת החשבוניות (10,000 ש"ח עד 31 במאי 2026, ואז 5,000 ש"ח החל מ-1 ביוני 2026), ודא שתגובת Morning API כוללת מספר הקצאה של רפורמת החשבוניות
7. הוספת פעולת מייל לשליחת קישור הקבלה ללקוח

תוצאה: כל תשלום קארדקום מוצלח מייצר אוטומטית קבלה ב-Morning ושולח אותה ללקוח במייל.

### דוגמה 2: סיכום מע"מ דו-חודשי

המשתמש אומר: "שלח לי סיכום כל החשבוניות בסוף כל תקופת מע"מ לרואה החשבון שלי"

פעולות:
1. יצירת Zap עם Schedule by Zapier, הרצה ב-10 לחודשים ספציפיים: מרץ, מאי, יולי, ספטמבר, נובמבר, ינואר
2. הוספת Webhooks by Zapier > Custom Request לקריאה ל-Morning API לשליפת מסמכים ב-2 החודשים הקודמים
3. הוספת Code by Zapier לחישוב סה"כ הכנסות, סה"כ מע"מ שנגבה, מספר חשבוניות
4. הוספת פעולת מייל לרואה חשבון עם טבלת סיכום RTL
5. הוספת Zapier Tables לארכיון סיכום התקופה

תוצאה: ב-10 במרץ, מאי, יולי, ספטמבר, נובמבר וינואר, נשלח מייל אוטומטי לרואה החשבון עם סיכום חשבוניות התקופה הדו-חודשית, 5 ימים לפני מועד ה-15 (או 9 ימים לפני מועד ה-19 המקוון).

### דוגמה 3: אישור תשלום בוואטסאפ דרך Twilio

המשתמש אומר: "שלח הודעת וואטסאפ בעברית ללקוחות כשהם משלמים"

פעולות:
1. הגדרת טריגר webhook לתשלום (IndicatorUrl של קארדקום או התראת טרנזילה)
2. הוספת Formatter לעיצוב סכום עם סיומת " ש\"ח"
3. הוספת Code by Zapier לעיצוב מספר טלפון: החלפת 0 מוביל ב-+972
4. הוספת פעולת Twilio > Send WhatsApp Message עם תבנית עברית מאושרת מראש
5. התבנית חייבת להיות מאושרת על ידי Meta מראש, למשל: "שלום {{1}}, קיבלנו תשלום בסך {{2}} ש\"ח. מספר אישור: {{3}}. תודה!"
6. מיפוי משתני תבנית: {{1}} = שם לקוח, {{2}} = סכום מעוצב, {{3}} = מזהה עסקה

חשוב: נדרש חשבון Twilio WhatsApp Business API עם תבניות מאושרות על ידי Meta. אינטגרציית וואטסאפ הרגילה של Zapier לא יכולה לשלוח הודעות ללקוחות.

תוצאה: הלקוח מקבל אישור תשלום בוואטסאפ בעברית תוך שניות מהתשלום, דרך תבנית מאושרת מראש.

### דוגמה 4: אוטומציית סוף שנה לעצמאים

המשתמש אומר: "אני צריך Zap שמכין סיכום שנתי של החשבוניות שלי להגשת מס"

פעולות:
1. יצירת Zap עם Schedule by Zapier, הרצה ב-15 בינואר
2. הוספת Webhooks by Zapier > Custom Request לקריאה ל-Morning API לשליפת מסמכים מכל השנה הקודמת
3. הוספת שלב Code by Zapier לחישוב סיכומים: הכנסות, הוצאות, מע"מ, ניכוי מס במקור
4. הוספת פעולת Gmail לשליחת סיכום RTL מעוצב לרואה החשבון
5. הוספת Zapier Tables לארכיון הסיכום השנתי

תוצאה: נתוני הכנת מס שנתיים נאספים אוטומטית ונשלחים לרואה החשבון בכל ינואר, הרבה לפני מועד ה-30 באפריל (או 31 במאי ואילך עם ייצוג רואה חשבון).

## מלכודות נפוצות

- **סכומי קארדקום הם בשקלים עשרוניים, לא באגורות**: קארדקום שולח `Amount` כערכים עשרוניים בשקלים (למשל 150.50 = 150.50 ש"ח). אסור לחלק ב-100. מפרט Swagger של Cardcom v11 מאשר שה-`Amount` בשקלים עשרוניים. חלוקה ב-100 תיצור חשבוניות ב-1/100 מהסכום הנכון.
- **כל מעבדי התשלומים הישראליים משתמשים בשקלים עשרוניים**: קארדקום, טרנזילה, Morning, Grow, iCount ו-EZcount שולחים ומקבלים סכומים בשקלים עשרוניים. אין צורך בהמרת אגורות לשקלים בשום מעבד.
- **אינטגרציית וואטסאפ הרגילה של Zapier מוגבלת לשליחה לעצמך בלבד**: אי אפשר לשלוח הודעות ללקוחות דרך אינטגרציית וואטסאפ המובנית של Zapier. היא תומכת ב-7 תבניות באנגלית בלבד. לוואטסאפ ללקוחות, השתמש ב-Twilio WhatsApp Business API, WATI או Respond.io, שכולם דורשים אימות Meta Business ותבניות הודעה מאושרות מראש.
- **Webhook של קארדקום הוא GET, לא POST**: ה-IndicatorUrl של קארדקום שולח בקשת GET עם פרמטרים (`InternalDealNumber`, `Amount`, `CardOwnerName` וכו'), לא גוף JSON POST. שם השדה למזהה העסקה הוא `InternalDealNumber`, לא `Transaction`.
- **אין אפליקציית Morning/חשבונית ירוקה מובנית ב-Zapier**: כל חיבורי Morning חייבים לעבור דרך "Webhooks by Zapier" עם REST API של Morning בכתובת `api.greeninvoice.co.il`.
- **אי-התאמת פורמט תאריך**: Zapier מגדיר כברירת מחדל פורמט אמריקאי (MM/DD/YYYY). מסמכים ישראליים משתמשים ב-DD/MM/YYYY. תמיד הוסף שלב עיצוב תאריך.
- **עברית בשלבי קוד**: ב-"Code by Zapier" (JavaScript או Python), מחרוזות עברית עובדות, אבל עברית בשמות משתנים תשבור את הקוד. שמות משתנים באנגלית בלבד.
- **סוגי מסמכים ב-Morning**: שאל את המשתמש איזה סוג מסמך הוא צריך. קבלה (400) מונפקת אחרי תשלום, חשבונית מס (305) לפני או בזמן מכירה, וחשבונית מס קבלה (320) משלבת את שניהם.
- **פורמט טלפון לוואטסאפ/Twilio**: מספרי נייד ישראליים חייבים קידומת +972 ללא ה-0 המוביל. השתמש ב-Code by Zapier: `phone.replace(/^0/, '+972')`.
- **שיעור מע"מ**: 18% מינואר 2025. סוכנים לפעמים משתמשים ב-17% המיושן.
- **Formatter לא מסיר סמני Unicode**: "Trim Whitespace" מסיר רווחים רגילים אך לא סמני RTL/LTR (U+200F, U+200E). השתמש בשלב Code by Zapier עם regex.
- **רפורמת החשבוניות 2026 משפיעה על אוטומציות, הסף יורד ב-1 ביוני 2026.** חשבוניות מס מעל הסף (10,000 ש"ח עד 31 במאי 2026, ואז 5,000 ש"ח החל מ-1 ביוני 2026) דורשות מספר הקצאה מרשות המסים. ודא שתהליך ה-API כולל שלב זה. שמור את הסף כמשתנה ב-Zap, לא כמספר קשיח.
- **מגבלות המסלול החינמי**: המסלול החינמי כולל Zaps ללא הגבלה אך רק שני שלבים (טריגר + פעולה אחת) ו-100 משימות בחודש. רוב האוטומציות העסקיות דורשות Zaps רב-שלביים, שזמינים החל ממסלול Starter ($19.99/חודש בשנתי). Professional ($49/חודש) נחוץ רק לפילטרים מתקדמים ולוגיקה מותנית.

## פתרון בעיות

### שגיאה: "Webhook לא מקבל נתונים מקארדקום"
סיבה: IndicatorUrl מוגדר שגוי או המסוף במצב בדיקה.
פתרון: ודא את ה-IndicatorUrl בהגדרות מסוף קארדקום. ודא שהמסוף במצב ייצור ולא sandbox. בצע תשלום אמיתי בסכום קטן (1 ש"ח). בדוק בהיסטוריית webhook של Zapier אם יש בקשות GET נכנסות (לא POST). אם משתמש ב-Catch Hook, ודא שהוא מקבל בקשות GET.

### שגיאה: "Morning API מחזיר 401 Unauthorized"
סיבה: מפתח API (JWT token) לא תקין, פג תוקף, או מעוצב לא נכון בכותרת Authorization.
פתרון: צור מפתח API חדש מדשבורד Morning (הגדרות > אינטגרציית API ב-app.greeninvoice.co.il). ודא שהטוקן נשלח כ-`Bearer <token>` בכותרת Authorization.

### שגיאה: "טקסט עברי מופיע משובש באימיילים"
סיבה: תבנית אימייל חסרה charset UTF-8 או כיוון RTL.
פתרון: עטוף תוכן עברי בתגית `<div dir="rtl">`. ודא שה-HTML כולל `<meta charset="UTF-8">`.

### שגיאה: "הודעת וואטסאפ לא נשלחת ללקוח"
סיבה: שימוש באינטגרציית וואטסאפ הרגילה של Zapier, שיכולה לשלוח רק לעצמך.
פתרון: עבור ל-Twilio WhatsApp Business API, WATI או Respond.io. אלו דורשים אימות Meta Business ותבניות הודעה מאושרות. אי אפשר לשלוח הודעות חופשיות ללקוחות דרך WhatsApp Business API.

### שגיאה: "מספר הקצאה חסר ברפורמת חשבוניות"
סיבה: חשבונית מעל סף רפורמת החשבוניות שנוצרה ללא מספר הקצאה מרשות המסים. הסף הוא 10,000 ש"ח עד 31 במאי 2026, ואז יורד ל-5,000 ש"ח ב-1 ביוני 2026.
פתרון: ודא שקריאת Morning API כוללת בקשה למספר הקצאה. Morning מטפל בזה אוטומטית עבור מסמכים שנוצרים דרך המערכת שלהם. בתהליכים ידניים, הוסף שלב שמבקש מספר הקצאה מרשות המסים לפני יצירת החשבונית. השווה את הסכום למשתנה ב-workflow שמכיל את הסף הנוכחי במקום למספר קשיח, מאחר שהסף מתוכנן לרדת שוב.

### שגיאה: "סכום קארדקום שגוי בחשבונית Morning"
סיבה: חלוקת סכום קארדקום ב-100 בטעות (עצה ישנה ושגויה).
פתרון: אסור לחלק סכומי קארדקום ב-100. שדה `Amount` של קארדקום כבר בשקלים עשרוניים (למשל 150.50 = 150.50 ש"ח). השתמש בערך כמות שהוא ביצירת מסמכי Morning.
